# 2.1 Анализ сложности алгоритмов

> *«Алгоритм считается хорошим не потому, что он работает быстро на вашем компьютере, а потому, что его скорость хорошо масштабируется с ростом размера входных данных.»*

---

- понимать смысл асимптотического анализа;
- оценивать временную и пространственную сложность алгоритмов;
- вычислять сложность циклов и рекурсивных алгоритмов;
- сравнивать различные алгоритмы;
- уверенно отвечать на вопросы про Big O на собеседовании.

---

# Зачем нужен анализ сложности?

Представим, что нужно найти максимальный элемент массива.

```python
nums = [4, 8, 1, 10, 6]
```

Самый простой алгоритм:

```python
maximum = nums[0]

for x in nums:
    if x > maximum:
        maximum = x
```

Работает быстро.

Но насколько быстро?

Если массив содержит:

- 10 элементов;
- 1000 элементов;
- 10 миллионов элементов,

время работы будет различаться.

Именно поэтому вместо абсолютного времени выполнения используется **асимптотическая сложность**.

---

# Что такое Big O?

Big O описывает, **как изменяется время работы алгоритма при увеличении размера входных данных**.

Пусть

```text
n
```

— размер входных данных.

Например,

- количество элементов массива;
- количество вершин графа;
- длина строки.

Нас интересует поведение алгоритма при очень больших значениях `n`.

---

# Почему не измерять время?

Допустим, алгоритм работает за

```
0.003 секунды
```

Что означает эта цифра?

Практически ничего.

Она зависит от:

- процессора;
- памяти;
- компилятора;
- интерпретатора;
- нагрузки системы.

Big O не зависит от компьютера.

Он показывает скорость роста количества операций.

---

# Основная идея

Рассмотрим цикл.

```python
for i in range(n):
    print(i)
```

Количество итераций

```
n
```

Следовательно,

временная сложность

```text
O(n)
```

---

Теперь

```python
for i in range(n):
    for j in range(n):
        print(i, j)
```

Внутренний цикл выполняется

```
n
```

раз для каждой итерации внешнего цикла.

Всего операций

```
n × n = n²
```

Следовательно,

```text
O(n²)
```

---

# Наиболее распространённые сложности

| Сложность | Название | Пример |
|-----------|----------|---------|
| O(1) | Константная | Доступ по индексу массива |
| O(log n) | Логарифмическая | Бинарный поиск |
| O(n) | Линейная | Проход по массиву |
| O(n log n) | Линейно-логарифмическая | Merge Sort |
| O(n²) | Квадратичная | Два вложенных цикла |
| O(2ⁿ) | Экспоненциальная | Полный перебор |
| O(n!) | Факториальная | Все перестановки |

---

# O(1)

Количество операций не зависит от размера данных.

Например,

```python
nums[10]
```

или

```python
stack.pop()
```

или

```python
stack.append(5)
```

Все эти операции выполняются за постоянное время.

---

# O(log n)

На каждой итерации количество элементов уменьшается примерно в два раза.

Самый известный пример —

бинарный поиск.

```
1024

↓

512

↓

256

↓

128

↓

...

↓

1
```

Количество шагов

```
log₂(n)
```

---

# O(n)

Алгоритм проходит по каждому элементу один раз.

Например,

```python
total = 0

for x in nums:
    total += x
```

---

# O(n log n)

Наиболее часто встречается в алгоритмах сортировки.

Например,

- Merge Sort;
- Heap Sort;
- Quick Sort (в среднем).

---

# O(n²)

Чаще всего появляется из-за двух вложенных циклов.

```python
for i in range(n):
    for j in range(n):
        ...
```

---

# O(2ⁿ)

Типичный пример —

полный перебор всех подмножеств.

Если

```
n = 20
```

то уже потребуется более миллиона вариантов.

---

# O(n!)

Факториальная сложность возникает при переборе всех перестановок.

Например,

задача коммивояжёра полным перебором.

Даже для небольших `n` такой алгоритм становится непрактичным.

---

# Big O игнорирует константы

Рассмотрим

```python
for i in range(5 * n):
    ...
```

Количество операций

```
5n
```

Однако

```
O(5n) = O(n)
```

Константа не влияет на порядок роста.

---

# Big O игнорирует младшие члены

Пусть

```
T(n) = n² + 5n + 100
```

При больших `n`

главную роль играет

```
n²
```

Следовательно,

```
O(n²)
```

---

# Несколько последовательных циклов

```python
for i in range(n):
    ...

for j in range(n):
    ...
```

Получаем

```
n + n = 2n
```

Следовательно,

```
O(n)
```

---

# Вложенные циклы

```python
for i in range(n):
    for j in range(n):
        ...
```

Получаем

```
n²
```

---

# Циклы разной длины

```python
for i in range(n):
    ...

for j in range(m):
    ...
```

Сложность

```
O(n + m)
```

---

# Пространственная сложность

Не менее важна,

чем временная.

Она показывает,

сколько дополнительной памяти использует алгоритм.

---

Например,

```python
maximum = nums[0]
```

Используется только одна переменная.

Следовательно,

```
O(1)
```

---

Теперь

```python
copy = []

for x in nums:
    copy.append(x)
```

Создаётся новый массив.

Следовательно,

```
O(n)
```

---

# Амортизированная сложность

Некоторые операции обычно работают быстро,

но иногда требуют дорогостоящего перераспределения памяти.

Например,

```python
lst.append(x)
```

Большинство вызовов

```
O(1)
```

Но иногда список увеличивает внутренний буфер,

что требует копирования всех элементов.

Тем не менее,

**амортизированная сложность**

остаётся

```
O(1)
```

Именно поэтому `append()` считается константной операцией.

---

# Что чаще всего спрашивают?

На собеседовании могут попросить оценить сложность кода.

Например,

```python
for i in range(n):
    print(i)
```

Ответ

```
O(n)
```

---

```python
for i in range(n):
    for j in range(n):
        print(i, j)
```

Ответ

```
O(n²)
```

---

```python
i = n

while i > 1:
    i //= 2
```

Ответ

```
O(log n)
```

---

```python
for i in range(n):
    j = 1
    while j < n:
        j *= 2
```

Внешний цикл

```
n
```

Внутренний

```
log n
```

Ответ

```
O(n log n)
```

# Что такое массив?

Массив — это структура данных, элементы которой располагаются **непрерывно в памяти**.

```
┌────┬────┬────┬────┬────┐
│  4 │  8 │ 15 │ 16 │ 23 │
└────┴────┴────┴────┴────┘
   0    1    2    3    4
```

Главное преимущество массива —

**быстрый доступ по индексу.**

---

# Почему доступ по индексу имеет сложность O(1)?

Каждый элемент массива имеет одинаковый размер.

Адрес элемента вычисляется по формуле

```
address = base + index × sizeof(element)
```

Поэтому не нужно проходить предыдущие элементы.

Например,

```python
nums[500000]
```

будет работать практически так же быстро, как

```python
nums[0]
```

# Dynamic Array

Во многих языках используется **динамический массив**.

В Python это

```python
list
```

В Java —

```java
ArrayList
```

В C++

```cpp
vector
```

---

# Как работает динамический массив?

Предположим,

есть массив ёмкостью

```
4
```

```
┌───┬───┬───┬───┐
│ 1 │ 2 │ 3 │   │
└───┴───┴───┴───┘
```

Добавляем новый элемент.

```
append(4)
```

Все хорошо.

Но если массив заполнен полностью,

```
┌───┬───┬───┬───┐
│ 1 │ 2 │ 3 │ 4 │
└───┴───┴───┴───┘
```

и мы выполняем

```python
append(5)
```

Python:

1. выделяет новый массив большей ёмкости;
2. копирует все элементы;
3. добавляет новый элемент.

---

# Почему append() считается O(1)?

На первый взгляд,

копирование всех элементов —

это

```
O(n)
```

Однако увеличение памяти происходит редко.

Большинство операций

```python
append()
```

выполняется без перераспределения памяти.

Поэтому говорят,

что

```
append()

↓

O(1) амортизированно
```

---

# Сложность операций списка

| Операция | Сложность |
|----------|----------:|
| `nums[i]` | O(1) |
| `append()` | O(1) амортизированно |
| `pop()` | O(1) |
| `insert(0, x)` | O(n) |
| `pop(0)` | O(n) |
| Поиск элемента | O(n) |

---

```
100 1 2 3
 ↑

```

Следовательно,

сложность

```
O(n)
```

# Two Pointers

Один из самых популярных паттернов интервью.

Используются два указателя,

которые двигаются по массиву.

```
0                     n

L ------------------> 

                <---- R
```

---

Самая простая задача.

```python
nums = [1,2,3,4,5]
```

Нужно развернуть массив.

Решение

```python
left = 0
right = len(nums)-1

while left < right:

    nums[left], nums[right] = nums[right], nums[left]

    left += 1
    right -= 1
```

---

# Когда использовать Two Pointers?

Если

- массив отсортирован;
- нужно искать пару элементов;
- нужно разворачивать массив;
- нужно двигаться с двух сторон.

---

# Пример

Задача Two Sum II.

```
1 2 4 6 9

target = 10
```

Используем два указателя.

```
L         R

1 2 4 6 9
```

Если сумма слишком маленькая —

двигаем левый.

Если слишком большая —

правый.

Получаем

```
O(n)
```

вместо

```
O(n²)
```

---

# Sliding Window

Второй по популярности паттерн.

Основная идея —

использовать окно,

которое постепенно двигается по массиву.

```
1 2 3 4 5 6 7

[-----]
```

---

Пример.

Максимальная сумма подмассива длины

```
k
```

Наивное решение

```
O(nk)
```

Sliding Window

```
O(n)
```

---

Идея.

Вместо повторного вычисления суммы

вычитаем элемент,

который покинул окно,

и добавляем новый.

---

# Когда использовать Sliding Window?

Практически всегда,

если встречаются слова

- подмассив;
- подстрока;
- максимальная длина;
- минимальная длина;
- непрерывный участок.

---

# Prefix Sum

Еще один очень популярный паттерн.

Создаем массив префиксных сумм.

```
nums

1 3 5 2

↓

prefix

0 1 4 9 11
```

Теперь сумма любого подмассива вычисляется за

```
O(1)
```

---

Например,

нужно найти сумму

```
[2...5]
```

Используем

```
prefix[r+1]-prefix[l]
```

---

# Когда использовать Prefix Sum?

Если нужно много раз вычислять суммы различных диапазонов.

---

# Как выбрать правильный паттерн?

| Если в условии... | Скорее всего нужен... |
|-------------------|-----------------------|
| Отсортированный массив | Two Pointers |
| Подстрока | Sliding Window |
| Подмассив | Sliding Window |
| Много запросов суммы | Prefix Sum |
| Поиск пары | Two Pointers |
| Максимальная длина | Sliding Window |

# Что такое Hash Table?

Hash Table (хеш-таблица) — структура данных, позволяющая хранить пары

```
ключ → значение
```

с очень быстрым доступом.

Например,

```python
student = {
    "name": "Ivan",
    "age": 22
}
```

---

# Главная идея

Вместо последовательного поиска

```
O(n)
```

используется специальная функция

```
hash(key)
```

которая преобразует ключ в индекс массива.

```
"apple"

↓

hash("apple")

↓

481273

↓

481273 % capacity

↓

5
```

Элемент сохраняется в ячейке номер 5.

---

# Как устроена Hash Table?

В основе лежит обычный массив.

```
0
1
2
3
4
5 → apple
6
7
8
9
```

Но индекс вычисляется автоматически.

---

# Hash Function

Hash-функция преобразует объект в целое число.

Например

```python
hash("apple")
```

или

```python
hash(42)
```

Полученное число используется для вычисления позиции элемента.

---

# Требования к hash-функции

Хорошая hash-функция должна:

- быть детерминированной;
- работать быстро;
- равномерно распределять ключи.

Если функция распределяет элементы плохо,

увеличивается количество коллизий.

---

# Что такое коллизия?

Коллизия возникает,

когда разные ключи попадают в одну и ту же ячейку.

Например

```
hash("cat") % 10 = 5

hash("dog") % 10 = 5
```

Оба элемента хотят занять одну позицию.

---

# Как решаются коллизии?

Существует два основных подхода.

---

## Separate Chaining

В каждой ячейке хранится список элементов.

```
5

↓

cat

↓

dog

↓

bird
```

Плюсы:

- простая реализация;
- легко добавлять элементы.

Минусы:

- дополнительная память.

---

## Open Addressing

Если ячейка занята,

ищется следующая свободная.

```
5 → cat

6 → dog

7 → bird
```

Именно этот подход используется в CPython.

---

# Почему dict работает быстро?

Большинство операций выполняются за

```
O(1)
```

потому что:

- хороший hash;
- низкая вероятность коллизий;
- автоматическое увеличение размера таблицы.

---

# Load Factor

Hash Table не должна быть слишком заполненной.

Используется коэффициент загрузки

```
Load Factor =
число элементов /
размер массива
```

Если таблица становится слишком заполненной,

она увеличивается.

---

# Rehashing

При увеличении размера таблицы

необходимо заново вычислить позиции всех элементов.

```
10 ячеек

↓

20 ячеек

↓

перераспределение всех ключей
```

Эта операция дорогая,

но происходит редко.

Поэтому вставка имеет

```
амортизированную O(1)
```

---

# Сложность операций

| Операция | Средний случай | Худший случай |
|----------|---------------:|--------------:|
| Поиск | O(1) | O(n) |
| Вставка | O(1) | O(n) |
| Удаление | O(1) | O(n) |

На практике худший случай встречается крайне редко.

---

# dict в Python

Словарь Python реализован как Hash Table.

Пример:

```python
user = {
    "name": "Alice",
    "age": 24
}
```

Получение значения

```python
user["name"]
```

работает примерно за

```
O(1)
```

---

# set в Python

Множество —

это Hash Table,

в которой хранятся только ключи.

Например,

```python
visited = {1, 2, 3}
```

или

```python
visited.add(10)

10 in visited
```

Все основные операции также имеют сложность

```
O(1)
```

---

# Какие объекты могут быть ключами?

Только hashable объекты.

Например,

можно использовать

```python
int

str

float

tuple
```

---

Нельзя использовать

```python
list

dict

set
```

Почему?

Потому что они изменяемые.

После изменения hash объекта изменился бы,

и словарь перестал бы работать корректно.

---

# Проверка hashability

Например,

```python
hash("abc")
```

работает.

А

```python
hash([1,2,3])
```

вызывает ошибку.

---

# Когда использовать dict?

Практически всегда,

если требуется:

- быстрый поиск;
- подсчёт частот;
- отображение ключ → значение;
- кэширование;
- запоминание посещённых вершин.

---

# Когда использовать set?

Если нужно:

- быстро проверять наличие элемента;
- удалять дубликаты;
- хранить множество уникальных объектов.

---

# Counter

```python
from collections import Counter

```

# defaultdict

Позволяет автоматически создавать значения.

Например,

```python
from collections import defaultdict

graph = defaultdict(list)
```

Теперь

```python
graph[5].append(10)
```

работает без проверки существования ключа.

---

# OrderedDict

До Python 3.7 использовался

```python
OrderedDict
```

для сохранения порядка элементов.

Начиная с Python 3.7

обычный

```python
dict
```

также сохраняет порядок вставки.

# Stack (стек)

Стек работает по принципу

> **LIFO (Last In — First Out)**

Последний добавленный элемент извлекается первым.

```
push(1)

↓

┌───┐
│ 1 │
└───┘

push(2)

↓

┌───┐
│ 2 │
├───┤
│ 1 │
└───┘

pop()

↓

┌───┐
│ 1 │
└───┘
```

---

# Где используется стек?

Стек применяется практически везде.

Например,

- рекурсия;
- вычисление выражений;
- проверка скобок;
- отмена действий (Undo);
- история браузера;
- DFS.

```
20
```

# Queue (очередь)

Очередь работает по принципу

> **FIFO (First In — First Out)**

Первый пришёл —

первый ушёл.

```
enqueue(1)

↓

1

enqueue(2)

↓

1 → 2

dequeue()

↓

2
```

---

# Где используется очередь?

Например,

- BFS;
- обработка запросов;
- очереди задач;
- планировщики;
- моделирование событий.

---

# Почему list плохо подходит?

Рассмотрим

```python
queue = []

queue.append(1)

queue.pop(0)
```

Операция

```python
pop(0)
```

имеет сложность

```
O(n)
```

потому что требуется сдвинуть все элементы.

---

# deque

В Python очередь реализуется через

```python
from collections import deque
```

---

Пример

```python
from collections import deque

queue.append(1)

queue.append(2)

queue.popleft()
```

Получаем

```
1
```

---

# Почему deque быстрее?

Deque реализован не как массив,

а как двусвязная структура блоков.

Поэтому операции

```
append()

appendleft()

pop()

popleft()
```

имеют сложность

```
O(1)
```

| Операция | Сложность |

# Deque

# 2.5 Linked List

> *«Связные списки редко используются в современных production-системах, однако на собеседованиях они встречаются регулярно, поскольку хорошо проверяют понимание указателей и работы памяти.»*

После изучения этого раздела вы сможете:

- использовать технику Fast & Slow Pointers;
- решать классические задачи интервью.

---

# Что такое Linked List?

Linked List (связный список) — структура данных, состоящая из узлов.

Каждый узел хранит:

- значение;
- ссылку на следующий элемент.

```
┌─────┬──────┐
│  10 │   •──┼────┐
└─────┴──────┘    │
                  ▼
            ┌─────┬──────┐
            │  20 │   •──┼────┐
            └─────┴──────┘    │
                              ▼
                        ┌─────┬──────┐
                        │  30 │ NULL │
                        └─────┴──────┘
```

---

# Node

Обычно узел выглядит так.

```python
class Node:

    def __init__(self, value):
        self.value = value
        self.next = None
```

---

# Head

Первый элемент списка называется

```
head
```

```
head

↓

10 → 20 → 30
```

Потеряв `head`, мы потеряем весь список.

---

# Почему Linked List не похож на массив?

В массиве

```
10 20 30 40
```

все элементы лежат подряд в памяти.

В Linked List

```
10

↓

20

↓

30

↓

40
```

элементы могут находиться где угодно.

Связь осуществляется ссылками.

---

# Преимущества Linked List

- простая вставка;
- простое удаление;
- не требуется непрерывная память;
- размер может увеличиваться постепенно.

---

# Недостатки

- нет доступа по индексу;
- больше памяти;
- хуже работает кэш процессора;
- медленный поиск.

---

# Сложность операций

| Операция | Linked List | Array |
|-----------|------------:|------:|
| Доступ по индексу | O(n) | O(1) |
| Поиск | O(n) | O(n) |
| Вставка в начало | O(1) | O(n) |
| Удаление начала | O(1) | O(n) |
| Вставка в конец* | O(n) | O(1) амортизированно |

\* Если нет указателя на хвост.

---

# Односвязный список

Каждый узел хранит

```

↓

```

```

```

---

# Двусвязный список

Каждый узел знает

- предыдущий элемент;
- следующий элемент.

```
NULL ← 1 ⇄ 2 ⇄ 3 → NULL
```

---

# Преимущества Doubly Linked List

Можно двигаться

в обе стороны.

Удаление узла происходит быстрее,

если ссылка на него уже известна.

---

# Обход списка

Самая частая операция.

```python
cur = head

while cur:

    print(cur.value)

    cur = cur.next
```

Сложность

```
O(n)
```

---

# Вставка в начало

```
head

↓

2 → 3
```

Добавляем

```
1
```

```
1 → 2 → 3
```

Нужно всего две операции:

```python
new.next = head

head = new
```

Сложность

```
O(1)
```

---

# Разворот списка

Одна из самых популярных задач.

```
1 → 2 → 3

↓

3 → 2 → 1
```

---

Идея.

Используем три указателя.

```
prev

cur

next
```

---

Алгоритм

```python
prev = None

cur = head

while cur:

    nxt = cur.next

    cur.next = prev

    prev = cur

    cur = nxt
```

После завершения

```
prev
```

становится новым

```
head
```

Очень популярный паттерн интервью.

```

```


`slow`

двигается

`fast`

по два.

```

```


дойдет до конца,

```

```

окажется в середине.

Сложность


```
fast

↓

1 → 2 → 3

↑       ↓

← ← ← ←
```

---

# 2.6 Binary Search

> *«Бинарный поиск — это не просто алгоритм поиска элемента. Это способ искать ответ в пространстве решений.»*

---

# Цели раздела

После изучения этого раздела вы сможете:

- понимать идею бинарного поиска;
- реализовывать классический Binary Search;
- находить Lower Bound и Upper Bound;
- применять Binary Search on Answer;
- распознавать задачи, в которых бинарный поиск является оптимальным решением.

---

# Что такое Binary Search?

Бинарный поиск — алгоритм поиска в **отсортированном массиве**.

Главная идея —

на каждом шаге отбрасывать половину элементов.

```
1 3 5 7 9 11 13 15

↓

1 3 5 7

или

9 11 13 15
```

---

# Почему Binary Search быстрый?

Пусть

```
n = 1 000 000
```

Линейный поиск

```
≈ 1 000 000 проверок
```

Бинарный поиск

```
≈ 20 проверок
```

Потому что

```
2²⁰ ≈ 1 000 000
```

---

# Когда можно использовать Binary Search?

Необходимы два условия.

- Данные отсортированы.

или

- Ответ можно проверить функцией

```
check(x)
```

которая является монотонной.

---

# Идея алгоритма

Имеем массив

```
1 4 6 8 10 15
```

Ищем

```
8
```

Берём середину.

```
6
```

Если

```
6 < 8
```

отбрасываем левую половину.

Если

```
6 > 8
```

отбрасываем правую.

Повторяем.

---

# Классическая реализация

```python
left = 0
right = len(nums) - 1

while left <= right:

    mid = (left + right) // 2

    if nums[mid] == target:
        return mid

    elif nums[mid] < target:
        left = mid + 1

    else:
        right = mid - 1

return -1
```

---

# Почему используется цикл

```python
left <= right
```

Пока существует хотя бы один кандидат.

После завершения

```
left > right
```

что означает,

что элемент отсутствует.

---

# Почему нельзя писать

```python
mid = (left + right) / 2
```

Во-первых,

получится число с плавающей точкой.

Во-вторых,

в некоторых языках возможно переполнение.

Безопасный вариант

```python
mid = left + (right - left) // 2
```

В Python переполнения нет,

но на интервью обычно показывают именно эту запись.

---

# Lower Bound

Lower Bound —

первый элемент,

который

```
>= target
```

Например,

```
1 2 2 2 4 5

target = 2
```

Ответ

```
индекс 1
```

---

# Upper Bound

Upper Bound —

первый элемент,

который

```
> target
```

Например,

```
1 2 2 2 4 5
```

Ответ

```
индекс 4
```

---

# Зачем нужны Lower и Upper Bound?

Например,

нужно узнать,

сколько раз встречается число.

```
count

=

upper_bound

-

lower_bound
```

---

# Binary Search on Answer

Самая интересная разновидность бинарного поиска.

Мы ищем

не элемент массива,

а ответ задачи.

---

Пример.

Есть

```
100 коробок
```

Нужно найти

минимальную грузоподъёмность машины.

Мы можем проверить

```
capacity = 20
```

Если подходит —

пробуем меньше.

Если нет —

увеличиваем.

---

Получается

```
False False False True True True
```

Ответ —

первая

```
True
```

---

# Монотонность

Binary Search on Answer

работает,

если существует переход

```
False False False True True True
```

или наоборот.

Без монотонности

бинарный поиск невозможен.

---

# Как понять,

что нужен Binary Search?

Если в условии встречаются слова

- минимальное значение;
- максимальное значение;
- наименьший ответ;
- наибольший ответ;
- можно ли...;
- существует ли...;

очень вероятно,

что требуется Binary Search on Answer.

---

# Типичные ошибки

## Ошибка 1

Использование

```python
while left < right
```

вместо

```python
while left <= right
```

для классического поиска.

---

## Ошибка 2

Забыть изменить

```
left
```

или

```
right
```

Получится бесконечный цикл.

---

## Ошибка 3

Использовать Binary Search

для неотсортированного массива.

---

## Ошибка 4

Неправильно выбирать середину.

## Что означает монотонность?

> После определённой точки значение функции перестаёт менять направление. Например: False → True.

## Когда Inorder возвращает отсортированный массив?

## Почему build_heap работает за O(n), а не за O(n log n)?

> Потому что большинство узлов находятся на нижних уровнях дерева и требуют минимального количества перестроений.

---

## Когда использовать Heap?

> Когда необходимо многократно получать минимальный или максимальный элемент.

---

## Когда использовать BFS?

> Когда требуется найти кратчайший путь в невзвешенном графе или выполнить обход по уровням.

---

## Почему Dijkstra использует Heap?

> Чтобы быстро выбирать вершину с минимальным текущим расстоянием.

---

## Когда нельзя использовать Dijkstra?

> При наличии отрицательных весов рёбер.

---

## Для чего нужен Union-Find?

> Для эффективного объединения множеств и проверки принадлежности элементов одной компоненте.

# Что такое Dynamic Programming?

Dynamic Programming (DP) — метод решения задач, в которых одна и та же подзадача возникает многократно.

Вместо повторных вычислений результат сохраняется и используется повторно.

---

# Когда применять DP?

Для применения динамического программирования обычно должны выполняться два условия.

## 1. Overlapping Subproblems

Подзадачи повторяются.

Например,

```
F(5)

↓

F(4)

↓

F(3)

↓

F(2)
```

При вычислении

```
F(5)
```

значение

```
F(3)
```

может вычисляться несколько раз.

---

## 2. Optimal Substructure

Оптимальное решение задачи можно построить из оптимальных решений её подзадач.

---

# Почему обычная рекурсия медленная?

Рассмотрим числа Фибоначчи.

```
F(5)

↓

F(4) + F(3)

↓

F(3) + F(2) + F(2) + F(1)

↓

...
```

Одни и те же значения вычисляются снова и снова.

Количество вызовов растёт экспоненциально.

```
O(2ⁿ)
```

---

# Memoization

Memoization —

сохранение результатов рекурсивных вызовов.

```python
memo = {}

def fib(n):

    if n in memo:
        return memo[n]

    ...
```

После вычисления

```
fib(30)
```

значение больше никогда не пересчитывается.

---

# Tabulation

Вместо рекурсии

строим решение снизу вверх.

```
F(0)

↓

F(1)

↓

F(2)

↓

...

↓

F(n)
```

---

# Memoization vs Tabulation

| Memoization | Tabulation |
|-------------|------------|
| Рекурсия | Цикл |
| Сверху вниз | Снизу вверх |
| Проще писать | Обычно быстрее |
| Использует стек | Не использует рекурсию |

---

# Как понять, что нужна DP?

На интервью часто встречаются слова

- максимальный;
- минимальный;
- количество способов;
- оптимальный;
- можно ли;
- наибольшая прибыль.

Это сильный сигнал,

что стоит подумать о Dynamic Programming.

# Как решать DP-задачи?

Практически любую задачу удобно решать по следующему алгоритму.

## Шаг 1

Определить состояние (State).

Что будет храниться в DP?

---

## Шаг 2

Определить переход (Transition).

Как вычисляется новое состояние?

---

## Шаг 3

Определить базовый случай.

---

## Шаг 4

Определить порядок вычислений.

---

# Пример

## Fibonacci

Состояние

```
dp[i]
```

означает

```
F(i)
```

Переход

```
dp[i]

=

dp[i-1]

+

dp[i-2]
```

---

# Пример

## Climbing Stairs

Можно подняться

на

```
1

или

2
```

ступеньки.

```
dp[i]

=

dp[i-1]

+

dp[i-2]
```

---

# House Robber

Есть дома.

Нельзя ограбить два соседних.

```
2 7 9 3 1
```

Переход

```
dp[i]

=

max(

dp[i-1],

dp[i-2] + nums[i]

)
```

---

# Coin Change

Есть монеты.

Нужно собрать сумму.

Состояние

```
dp[i]
```

—

минимальное число монет

для суммы

```
i
```

---

# Longest Increasing Subsequence

Нужно найти

самую длинную возрастающую подпоследовательность.

```
10 9 2 5 3 7 101
```

Ответ

```
2 3 7 101
```

---

# Longest Common Subsequence

Даны две строки.

```
ABCDEF

ACEF
```

Необходимо найти

самую длинную общую подпоследовательность.

---

# Knapsack

Классическая задача.

Есть предметы

с весом

и стоимостью.

Нужно выбрать

максимальную стоимость,

не превышая допустимый вес.

# Interview Summary

Перед собеседованием обязательно запомните:

- Dynamic Programming применяется, когда подзадачи повторяются.
- Основные подходы — Memoization и Tabulation.

- Важно уметь оптимизировать память.
- Не каждая рекурсивная задача требует DP.
- На интервью чаще всего встречаются задачи **House Robber**, **Coin Change**, **LIS**, **LCS**, **Knapsack**, **Climbing Stairs**.
