# 1.2 Объекты, память и копирование

> *«Если вы понимаете модель памяти Python, то автоматически понимаете половину языка.»*

После изучения этого раздела вы сможете:

- понимать разницу между объектом и ссылкой;

- объяснять работу `id()`, `is` и `==`;

# Всё в Python — объект

Практически всё, с чем вы работаете в Python, является объектом.

Например:

```python
10
```

```python
3.14
```

```python
"Hello"
```

```python
[1, 2, 3]
```

```python
{"x": 1}
```

```python
def foo():
    pass
```

Все перечисленные сущности являются объектами.

---

# Что хранит переменная?

Это один из самых популярных вопросов на собеседованиях.

Большинство начинающих отвечает:

> Переменная хранит значение.

Это неверно.

Переменная хранит **ссылку на объект**.

Например,

```python
x = [1, 2, 3]
```

Концептуально память можно представить так:

```
x
│
▼
┌───────────────┐
│   [1, 2, 3]   │
└───────────────┘
```

---

# Что происходит при присваивании?

Рассмотрим пример.

```python
a = [1, 2, 3]
b = a
```

Очень многие считают, что создаётся копия списка.

На самом деле память выглядит следующим образом.

```
a ─────┐
       │
       ▼
┌───────────────┐
│   [1, 2, 3]   │
└───────────────┘
       ▲
       │
b ─────┘
```

Объект один.

Ссылок две.

---

## Assignment никогда не копирует объект

Запомните главное правило Python.

> Оператор `=` **никогда не копирует объект**.

Он лишь копирует ссылку.

Именно поэтому следующий код работает именно так.

```python
a = [1, 2]
b = a

a.append(3)

print(b)
```

Ответ:

```python
[1, 2, 3]
```

---

# Объект и ссылка — разные понятия

Необходимо различать три сущности.

```
Переменная
      │
      ▼
   Ссылка
      │
      ▼
    Объект
```

Переменная хранит ссылку.

Ссылка указывает на объект.

Объект содержит данные.

---

# Что такое `id()`?

Функция

```python
id(obj)
```

возвращает идентификатор объекта.

В реализации CPython этот идентификатор обычно совпадает с адресом объекта в памяти.

Например,

```python
a = [1]

print(id(a))
```

Возможный вывод:

```
4378560000
```

Важно помнить, что стандарт Python гарантирует лишь уникальность идентификатора во время жизни объекта. То, что в CPython он совпадает с адресом памяти, — особенность реализации, а не требование языка.

---

# Mutable и Immutable объекты

Все встроенные типы Python можно условно разделить на две группы.

## Mutable

Объект можно изменить после создания.

Примеры:

- `list`
- `dict`
- `set`
- `bytearray`

Например,

```python
a = [1, 2]

a.append(3)
```

После выполнения программы объект остаётся тем же самым.

Изменяется только его содержимое.

---

## Immutable

Объект изменить нельзя.

Любая операция создаёт новый объект.

Примеры:

- `int`
- `float`
- `bool`
- `str`
- `tuple`
- `bytes`
- `frozenset`

Например,

```python
a = 10
a += 1
```

Создаётся новый объект `11`.

---

# Проверим это

```python
a = [1, 2]

print(id(a))

a.append(3)

print(id(a))
```

Вывод:

```
4384956800
4384956800
```

Идентификатор не изменился.

Следовательно, объект тот же.

Теперь рассмотрим число.

```python
a = 10

print(id(a))

a += 1

print(id(a))
```

Идентификаторы будут различаться.

Следовательно, создан новый объект.

---

# Особенность `tuple`

Очень популярный вопрос на собеседованиях.

```python
x = (1, [2])

x[1].append(3)

print(x)
```

Ответ:

```python
(1, [2, 3])
```

Почему это возможно?

Потому что `tuple` гарантирует неизменность своих ссылок.

Но объекты, на которые указывают эти ссылки, вполне могут быть mutable.

---

# `==` и `is`

Эти операторы решают разные задачи.

## `==`

Сравнивает значения объектов.

```python
a = [1, 2]
b = [1, 2]

print(a == b)
```

Ответ:

```python
True
```

---

## `is`

Сравнивает идентичность объектов.

```python
a = [1, 2]
b = [1, 2]

print(a is b)
```

Ответ:

```python
False
```

Потому что это два разных объекта.

---

## Когда использовать `is`?

Практически всегда только при сравнении с `None`.

Правильно:

```python
if value is None:
    ...
```

Неправильно:

```python
if value == None:
    ...
```

Причина заключается в том, что `==` вызывает метод `__eq__()`, тогда как `is` проверяет идентичность объектов.

---

# Поверхностное копирование

Рассмотрим вложенный список.

```python
import copy

a = [[1], [2]]

b = copy.copy(a)
```

После копирования создаётся новый внешний список.

Однако внутренние списки остаются общими.

```
a                b

┌──────┐      ┌──────┐
│ • •  │      │ • •  │
└─┼─┼──┘      └─┼─┼──┘
  │ │           │ │
  ▼ ▼           ▼ ▼
 [1][2]       [1][2]
```

Проверим.

```python
b[0].append(100)

print(a)
```

Ответ:

```python
[[1, 100], [2]]
```

---

# Глубокое копирование

Теперь используем

```python
copy.deepcopy()
```

```python
import copy

a = [[1], [2]]

b = copy.deepcopy(a)
```

Теперь копируются абсолютно все вложенные объекты.

Изменение копии никак не влияет на оригинал.

---

# Почему `deepcopy` работает медленнее?

Потому что ему приходится:

- рекурсивно обходить всю структуру объекта;
- создавать новые объекты;
- отслеживать уже скопированные объекты с помощью словаря `memo`;
- корректно обрабатывать циклические ссылки.

Именно поэтому `deepcopy()` значительно дороже обычного копирования.

# Функции — это объекты

В Python функции являются объектами первого класса (*first-class objects*).

Это означает, что функцию можно:

- сохранить в переменной;
- передать как аргумент;
- вернуть из другой функции;
- сохранить в контейнере.

Например,

```python
def square(x):
    return x * x

f = square

print(f(5))
```

Результат

```python
25
```

Переменная `f` теперь содержит ссылку на объект-функцию.

---

# Передача аргументов

Рассмотрим функцию.

```python
def foo(x):
    x.append(10)

a = [1, 2]

foo(a)

print(a)
```

Ответ

```python
[1, 2, 10]
```

Почему?

Потому что функция получает **ссылку на объект**, а не его копию.

Это называют

> **Pass by Object Reference**

или

> **Call by Sharing**

Важно понимать:

Python **не использует** ни *pass-by-value*, ни *pass-by-reference* в классическом понимании.

Он передает ссылку на объект.

---

# Изменение объекта и переприсваивание

Сравним два примера.

### Изменение объекта

```python
def foo(x):
    x.append(3)

a = [1, 2]

foo(a)

print(a)
```

Ответ

```python
[1, 2, 3]
```

---

### Переприсваивание

```python
def foo(x):
    x = [100]

a = [1, 2]

foo(a)

print(a)
```

Ответ

```python
[1, 2]
```

Во втором примере изменяется не объект, а локальная переменная `x`.

# Параметры и аргументы

Не стоит путать эти понятия.

```python
def add(a, b):
    return a + b

add(3, 5)
```

Здесь

- `a`, `b` — **параметры** (*parameters*);
- `3`, `5` — **аргументы** (*arguments*).

# Позиционные аргументы

По умолчанию аргументы передаются по порядку.

```python

greet("Иван", 20)
```

# Именованные аргументы

Аргументы можно передавать по имени.

```python
greet(age=20, name="Иван")
```

Это делает код более читаемым и позволяет менять порядок аргументов.

```python
def power(x, p=2):
    return x ** p
```

# Самая известная ловушка Python


# Почему это происходит?

Значения аргументов по умолчанию вычисляются **один раз** — в момент определения функции.

Поэтому список создается только один раз и используется при каждом вызове.

---

# Правильный вариант

```python
def foo(lst=None):

        lst = []

    lst.append(1)
    return lst
```

Такой вариант считается правильным и рекомендуется PEP 8.

# *args

Иногда количество аргументов заранее неизвестно.

```python
*args
```

```python
def total(*args):

print(total(1, 2, 3, 4))
```


Внутри функции `args` имеет тип `tuple`.

```python
**kwargs

```

```python
{'name': 'Ivan', 'age': 20}
```

Внутри функции `kwargs` — обычный словарь.

# Распаковка аргументов

Символы `*` и `**` используются не только при объявлении функции, но и при её вызове.

```python
nums = [1, 2, 3]

```

Результат

```python
1 2 3
```

```python
data = {
```

Эквивалентно

# Правило LEGB

Когда Python встречает имя переменной, он ищет его в следующем порядке.

1. **L — Local**
2. **E — Enclosing**
3. **G — Global**
4. **B — Built-in**

Эта последовательность называется правилом **LEGB**.

---

# Local

```python
x = 10

def foo():
    x = 20
    print(x)

foo()
```

Ответ

```python
20
```

Используется локальная переменная.

---

# Global

```python
x = 10

def foo():
    print(x)

foo()
```

Ответ

```python
10
```

Локальной переменной нет, поэтому используется глобальная.

---

# global

Для изменения глобальной переменной используется ключевое слово

```python
global
```

```python
x = 0

def inc():
    global x
    x += 1

inc()

print(x)
```

Ответ

```python
1
```

Использовать `global` рекомендуется только при реальной необходимости.

---

# nonlocal

Используется для изменения переменной внешней функции.

```python
def outer():
    x = 10

    def inner():
        nonlocal x
        x += 1

    inner()

    return x

print(outer())
```

Ответ

```python
11
```

```python
def multiplier(k):

    return mul

times2 = multiplier(2)

print(times2(5))
```

Ответ

```python
10
```

Переменная `k` продолжает существовать даже после завершения работы `multiplier()`.

# Декораторы

Декоратор — это функция, которая принимает другую функцию и возвращает новую.

Простейший пример

```python
def decorator(func):

        print("Before")

def hello():

```

Результат

```text
Before
Hello
After
```

На практике декораторы используются для:

- логирования;
- измерения времени выполнения;
- проверки прав доступа;
- кеширования;
- обработки исключений.

# Iterable и Iterator

Эти понятия часто путают.

## Iterable

**Iterable** — объект, по которому можно пройти в цикле `for`.

Например,

```python
lst = [1, 2, 3]

for x in lst:
    print(x)
```

Список является iterable.

Другие примеры:

- list
- tuple
- dict
- set
- str
- range

---

## Iterator

**Iterator** — объект, который возвращает следующий элемент последовательности.

Он реализует два метода:

```python
__iter__()
```

и

```python
__next__()
```

---

Получить итератор можно с помощью функции

```python
iter()
```

Например

```python
nums = [1, 2, 3]

it = iter(nums)

print(next(it))
print(next(it))
print(next(it))
```

Ответ

```python
1
2
3
```

Следующий вызов

```python
next(it)
```

приведёт к исключению

```python
StopIteration
```

---

# Iterator Protocol

Любой объект считается итератором, если реализует два метода.

```python
__iter__()
```

Возвращает сам объект.

---

```python
__next__()
```

Возвращает следующий элемент.

Если элементы закончились,

генерируется

```python
StopIteration
```

---

# Как работает цикл for?

Конструкция

```python
for x in iterable:
    print(x)
```

примерно эквивалентна следующему коду.

```python
it = iter(iterable)

while True:
    try:
        x = next(it)
        print(x)
    except StopIteration:
        break
```

Именно поэтому любой iterable можно использовать в цикле `for`.

---

# Генераторы

Генератор — это специальный вид итератора.

Главное отличие заключается в том, что генератор **не хранит все элементы сразу**.

Он создаёт их по мере необходимости.

---

# yield

Рассмотрим пример.

```python
def numbers():
    yield 1
    yield 2
    yield 3
```

Теперь

```python
gen = numbers()

print(next(gen))
print(next(gen))
print(next(gen))
```

Ответ

```python
1
2
3
```

---

# Как работает yield?

Обычная функция после выполнения

```python
return
```

завершается.

Например

```python
def foo():
    return 1
```

После возврата функция прекращает существование.

---

Оператор

```python
yield
```

работает иначе.

После выполнения

```python
yield
```

состояние функции сохраняется.

Следующий вызов

```python
next()
```

продолжает выполнение **с того места, где функция остановилась**.

---

# Почему генераторы экономят память?

Рассмотрим пример.

```python
nums = list(range(1_000_000))
```

Создаются один миллион объектов.

Теперь

```python
nums = (x for x in range(1_000_000))
```

Создаётся только генератор.

Числа будут вычисляться только тогда, когда они действительно понадобятся.

Поэтому генераторы особенно полезны при обработке больших файлов, потоков данных и датасетов.

---

# Generator Expression

Аналогично list comprehension существует generator expression.

```python
gen = (x * x for x in range(5))
```

В отличие от

```python
lst = [x * x for x in range(5)]
```

генератор не вычисляет значения заранее.

---

# Когда использовать генераторы?

Используйте генераторы, если:

- последовательность очень большая;
- элементы нужны по одному;
- важно экономить память.

Используйте список, если:

- нужен случайный доступ по индексу;
- данные будут использоваться несколько раз;
- размер последовательности небольшой.

---

# Менеджеры контекста

Менеджер контекста управляет выделением и освобождением ресурсов.

Самый известный пример.

```python
with open("file.txt") as f:
    data = f.read()
```

После выхода из блока

```python
with
```

файл автоматически закрывается.

---

Менеджер контекста реализует два метода.

```python
__enter__()
```

```python
__exit__()
```

---

# Зачем нужен with?

Без него

```python
f = open("file.txt")

data = f.read()

f.close()
```

Если между `open()` и `close()` возникнет исключение,

файл может остаться открытым.

Менеджер контекста гарантирует корректное освобождение ресурсов.

---

# GIL

**Global Interpreter Lock** — глобальная блокировка интерпретатора CPython.

Она гарантирует, что в один момент времени только один поток выполняет Python-байткод.

---

# Зачем нужен GIL?

Основная причина — упрощение работы с памятью и подсчётом ссылок.

Без GIL пришлось бы синхронизировать доступ к каждому объекту Python.

Это значительно усложнило бы реализацию интерпретатора.

---

# Следствие GIL

Потоки в CPython **не могут одновременно выполнять Python-код** на нескольких ядрах.

Это означает, что `threading` не ускоряет CPU-bound задачи.

---

# CPU-bound и I/O-bound

## CPU-bound

Основное время тратится на вычисления.

Примеры:

- обучение модели;
- обработка изображений;
- вычисление матриц;
- поиск простых чисел.

Для таких задач лучше использовать

```python
multiprocessing
```

---

## I/O-bound

Основное время тратится на ожидание.

Например:

- работа с сетью;
- чтение файлов;
- запросы к базе данных;
- HTTP API.

Для таких задач хорошо подходит

```python
threading
```

или

```python
asyncio
```

---

# Threading

Позволяет выполнять несколько потоков.

Преимущества:

- простой API;
- подходит для I/O-bound задач.

Недостаток:

- ограничение GIL.

---

# Multiprocessing

Создаёт отдельные процессы.

Каждый процесс имеет собственный интерпретатор Python.

Следовательно,

каждый процесс имеет собственный GIL.

Именно поэтому multiprocessing позволяет использовать несколько ядер процессора.

Недостатки:

- больший расход памяти;
- более дорогой запуск процессов;
- необходимость сериализации объектов.

---

# Asyncio

Основан на событийном цикле.

Используется для большого количества одновременно выполняющихся операций ввода-вывода.

Например:

```python
async def download():
    ...

await download()
```

Преимущества:

- очень эффективен для большого количества сетевых запросов;
- минимальные накладные расходы.

Недостаток:

- не ускоряет CPU-bound задачи.

---

# Что выбрать?

| Задача | Решение |
|---------|----------|
| Вычисления | `multiprocessing` |
| HTTP-запросы | `asyncio` |
| Работа с файлами | `threading` или `asyncio` |
| Обучение моделей | `multiprocessing` |
| Обработка сетевых соединений | `asyncio` |

# Часто задаваемые вопросы

## 1. Что хранит переменная в Python?

**Ответ**

Переменная хранит ссылку на объект, а не сам объект и не его значение.

---

## 2. Что делает оператор `=`?

**Ответ**

Копирует ссылку на объект.

Никакого копирования самого объекта не происходит.

---

## 3. Чем отличается mutable от immutable объекта?

**Ответ**

Mutable-объект можно изменить после создания.

Immutable-объект изменить нельзя — любая операция создаёт новый объект.

---

## 4. Что делает `id()`?

**Ответ**

Возвращает идентификатор объекта.

В CPython он обычно совпадает с адресом объекта в памяти.

---

## 5. Чем отличается `==` от `is`?

**Ответ**

`==`

сравнивает значения.

`is`

сравнивает идентичность объектов.

---

## 6. Когда использовать `is`?

Практически всегда только для сравнения с

```python
None
```

Например

```python
if x is None:
```

---

## 7. Что такое shallow copy?

Создаётся новый внешний контейнер.

Вложенные объекты остаются общими.

---

## 8. Что такое deepcopy?

Рекурсивное копирование всей структуры объекта.

---

## 9. Почему deepcopy медленнее?

Потому что рекурсивно копирует все вложенные объекты и отслеживает уже скопированные объекты.

---

## 10. Как Python передаёт аргументы в функцию?

Python передаёт ссылку на объект.

Это называется

> Pass by Object Reference

или

> Call by Sharing.

---

## 11. Почему нельзя использовать mutable-объекты как значения по умолчанию?

Потому что значение по умолчанию вычисляется один раз при определении функции.

---

## 12. Что такое LEGB?

Порядок поиска имени:

1. Local
2. Enclosing
3. Global
4. Built-in

---

## 13. Что такое замыкание?

Функция, которая сохраняет доступ к переменным внешней области видимости.

---

## 14. Что делает `yield`?

Приостанавливает выполнение функции и сохраняет её состояние.

---

## 15. Чем генератор отличается от списка?

Генератор создаёт элементы по мере необходимости и не хранит всю последовательность в памяти.

---

## 16. Что такое GIL?

Global Interpreter Lock.

Позволяет только одному потоку одновременно выполнять Python-байткод.

---

## 17. Почему threading не ускоряет вычисления?

Из-за GIL.

---

## 18. Когда использовать multiprocessing?

Для CPU-bound задач.

---

## 19. Когда использовать asyncio?

Для большого количества операций ввода-вывода.

---

## 20. Что такое декоратор?

Функция, принимающая другую функцию и возвращающая новую.
