# Что такое NLP?

**Natural Language Processing (NLP)** — область искусственного интеллекта, занимающаяся обработкой естественного языка.

Под естественным языком понимаются языки, на которых общаются люди: русский, английский, немецкий, китайский и многие другие.

Главная цель NLP — научить компьютер понимать, анализировать и генерировать человеческую речь и текст.

---

# Почему NLP — сложная задача?

На первый взгляд текст кажется достаточно простой формой данных. Однако язык обладает рядом особенностей, которые делают его обработку значительно сложнее, чем может показаться.

Во-первых, одно и то же можно сказать десятками разных способов.

Например:

```
Мне нравится эта книга.

Эта книга мне очень понравилась.

Я в восторге от этой книги.
```

Несмотря на разные слова и структуру предложений, смысл практически одинаков.

Во-вторых, порядок слов имеет огромное значение.

Например:

```
Собака укусила человека.
```

и

```
Человек укусил собаку.
```

используют почти одинаковый набор слов, но описывают совершенно разные ситуации.

Наконец, значение слова часто определяется только контекстом.

Например, слово **«ключ»** может означать:

- инструмент для открытия замка;
- источник воды;
- решение задачи;
- музыкальный ключ.

Без окружающего контекста понять правильный смысл невозможно.

---

# Почему компьютеру трудно понимать текст?

Компьютер умеет работать только с числами.

Для человека предложение

> «Я люблю машинное обучение.»

имеет очевидный смысл.

Для компьютера это всего лишь последовательность символов.

Поэтому первая задача любой NLP-системы — преобразовать текст в числовое представление, с которым сможет работать алгоритм машинного обучения.

Практически вся история NLP — это история поиска всё более удачных способов такого представления.

---

# Какие задачи решает NLP?

Сегодня NLP используется практически во всех современных цифровых продуктах.

Наиболее распространённые задачи:

- классификация текста;
- анализ тональности;
- машинный перевод;
- генерация текста;
- поиск информации;
- ответы на вопросы;
- суммаризация документов;
- извлечение информации (Information Extraction);
- распознавание именованных сущностей (NER);
- диалоговые системы и чат-боты.

Практически все современные голосовые ассистенты и большие языковые модели используют комбинацию нескольких подобных задач.

---

# Как развивалось NLP?

Эволюцию NLP удобно рассматривать как последовательность попыток всё лучше представить смысл текста.

Каждый новый этап решал проблемы предыдущего.

```
One-Hot Encoding

Bag of Words
        ↓
TF-IDF
        ↓
Word2Vec / GloVe / FastText
        ↓
RNN
        ↓
LSTM / GRU
        ↓
Attention
        ↓
Transformer
        ↓
BERT / GPT / современные LLM
```

Именно этот путь мы пройдём в данной главе.

# 7.1 Представление текста

> *«Первая проблема, которую пришлось решить в NLP, возникла задолго до появления нейронных сетей: как представить человеческий язык в виде чисел?»*

# Цели раздела

После изучения этого раздела вы сможете:

- понимать, зачем текст необходимо преобразовывать в числа;
- знать классические способы представления текста;
- понимать преимущества и недостатки One-Hot Encoding, Bag of Words и TF-IDF;
- объяснять, почему появились Word Embeddings;
- отвечать на вопросы интервью по классическому NLP.

---

# Почему текст нужно преобразовывать в числа?

Любой алгоритм машинного обучения работает исключительно с числами.

Изображение можно представить как матрицу пикселей. Аудио — как последовательность амплитуд сигнала. Но текст сам по себе не является числовым объектом.

Например, предложение

> **«Я люблю машинное обучение.»**

для человека имеет очевидный смысл, а для компьютера представляет собой лишь последовательность символов. Пока мы не превратим эти символы в числа, никакая модель не сможет с ними работать.

Поэтому практически любая NLP-система начинается с преобразования текста в некоторое числовое представление.

---

# Каким должно быть хорошее представление текста?

Идеальное представление должно удовлетворять нескольким требованиям.

Оно должно:

- быть числовым;
- сохранять смысл текста;
- учитывать сходство между словами;
- быть компактным;
- эффективно обрабатываться моделью.

Первые методы решали лишь часть этих задач.

---

# One-Hot Encoding

## 📖 Что это?

**One-Hot Encoding** — самый простой способ представить слово в виде вектора.

Пусть словарь состоит из четырёх слов:

```
кот
собака
машина
дом
```

Тогда каждому слову соответствует собственная позиция.

| Слово | Вектор |
|-------|---------|
| кот | [1, 0, 0, 0] |
| собака | [0, 1, 0, 0] |
| машина | [0, 0, 1, 0] |
| дом | [0, 0, 0, 1] |

Каждое слово отличается только положением единицы.

---

## Почему этот метод работает?

Потому что модель получает уникальный идентификатор для каждого слова.

На этом преимущества практически заканчиваются.

---

## Недостаток №1. Огромная размерность

Если словарь содержит миллион слов, каждый вектор тоже будет иметь миллион элементов.

При этом почти все значения будут равны нулю.

Такое представление называют **разреженным (Sparse Representation)**.

---

## Недостаток №2. Нет понимания смысла

Для One-Hot слова

```
кот
```

и

```
собака
```

настолько же различны, как

```
кот
```

и

```
квантовая механика
```

Модель не знает, что кошка и собака — похожие объекты.

---

> 🎯 Главное

One-Hot Encoding позволяет различать слова, но совершенно не понимает их смысл.

---

### 🎤 Вопрос с интервью

**Почему One-Hot Encoding практически не используется в современных NLP-моделях?**

Потому что такие векторы имеют огромную размерность, содержат очень много нулей и не отражают семантическую близость слов.

---

# Что такое разреженный вектор?

Этот термин часто встречается не только в NLP.

**Разреженный (Sparse) вектор** — вектор, большинство элементов которого равны нулю.

Например,

```
[0, 0, 0, 0, 0, 1, 0, 0, 0]
```

В противоположность ему существует **плотный (Dense) вектор**, в котором большинство элементов содержат полезную информацию.

Например,

```
[0.43, -1.12, 0.08, 0.77]
```

Практически все современные нейронные сети работают именно с плотными представлениями.

---

# Bag of Words (BoW)

## 📖 Что это?

**Bag of Words** представляет документ как набор слов без учёта их порядка.

Модель считает, сколько раз каждое слово встретилось в тексте.

Например, документ

```
Кот любит молоко.
Кот любит рыбу.
```

превращается в словарь

| Слово | Количество |
|--------|------------|
| кот | 2 |
| любит | 2 |
| молоко | 1 |
| рыбу | 1 |

Получается вектор

```
[2, 2, 1, 1]
```

---

## Почему метод называется "мешок слов"?

Представьте, что все слова предложения положили в мешок и хорошо перемешали.

Мы знаем, какие слова присутствуют, но полностью теряем информацию об их порядке.

Именно поэтому метод получил название **Bag of Words**.

---

# Главная проблема Bag of Words

Рассмотрим два предложения.

```
Собака укусила человека.
```

```
Человек укусил собаку.
```

Bag of Words построит практически одинаковые представления, хотя смысл предложений противоположен.

Модель знает только частоту слов, но не знает, кто кого укусил.

---

### Production Notes

Несмотря на возраст, Bag of Words до сих пор используется как простой baseline для задач классификации текста и тематического анализа.

---

### 🎤 Вопрос с интервью

**Какой главный недостаток Bag of Words?**

Он полностью игнорирует порядок слов.

---

# N-граммы

Bag of Words потерял информацию о порядке слов.

Одним из первых способов исправить эту проблему стали **N-граммы**.

---

## 📖 Что это?

N-грамма — последовательность из N соседних слов.

Например, предложение

```
Я люблю машинное обучение.
```

содержит:

**Униграммы**

```
Я
люблю
машинное
обучение
```

**Биграммы**

```
Я люблю
люблю машинное
машинное обучение
```

**Триграммы**

```
Я люблю машинное
люблю машинное обучение
```

---

## Что дают N-граммы?

Теперь модель начинает учитывать локальный порядок слов.

Например,

```
машинное обучение
```

становится отдельным признаком, отличным от слов "машинное" и "обучение" по отдельности.

Это позволяет лучше учитывать устойчивые словосочетания.

---

## Почему N-граммы не решили проблему?

Количество возможных N-грамм очень быстро растёт.

Даже для среднего словаря число различных биграмм и триграмм становится огромным.

Поэтому такие модели быстро начинают потреблять слишком много памяти.

---

# TF-IDF

## 📖 Что это?

**TF-IDF (Term Frequency – Inverse Document Frequency)** — развитие идеи Bag of Words.

Главное наблюдение очень простое.

Не все слова одинаково важны.

---

## Интуиция

Представьте два слова.

```
и
```

и

```
рентгенография
```

Слово "и" встречается почти в каждом документе.

Оно практически не помогает понять, о чём текст.

Слово "рентгенография" встречается редко.

Если оно появилось в документе, это уже очень информативный признак.

Именно эту идею использует TF-IDF.

---

## Из чего состоит TF-IDF?

### TF (Term Frequency)

Показывает, насколько часто слово встречается внутри конкретного документа.

---

### IDF (Inverse Document Frequency)

Показывает, насколько редко слово встречается во всём корпусе документов.

Чем реже слово встречается, тем больший вес оно получает.

---

## Что получается в итоге?

TF-IDF автоматически уменьшает влияние слов вроде

```
и
в
на
это
```

и усиливает влияние действительно информативных терминов.

Благодаря этому качество моделей обычно оказывается выше, чем при использовании обычного Bag of Words.

---

### Production Notes

TF-IDF до сих пор активно применяется:

- в информационном поиске;
- в поисковых системах;
- в простых системах рекомендаций документов;
- как baseline в задачах классификации текста.

---

### 🎤 Вопрос с интервью

**Чем TF-IDF отличается от Bag of Words?**

Bag of Words считает все слова одинаково важными.

TF-IDF автоматически уменьшает вес слишком частых слов и усиливает влияние редких информативных терминов.

---

# Почему классические методы перестали устраивать?

Все рассмотренные методы имеют общие ограничения.

Во-первых, они не понимают смысл слов.

Во-вторых, они практически не учитывают контекст.

В-третьих, их размерность напрямую зависит от размера словаря.

Самая серьёзная проблема заключается в том, что слова существуют независимо друг от друга.

Для модели слова

```
кошка
```

и

```
кот
```

ничем не ближе, чем

```
кошка
```

и

```
автомобиль
```

Очевидно, хотелось получить представление, в котором похожие слова действительно находятся рядом.

Именно эта идея привела к появлению **Word Embeddings**.

# Сравнение методов

| Метод | Учитывает порядок слов | Понимает смысл слов | Размер представления |
|--------|------------------------|---------------------|----------------------|
| One-Hot Encoding | ❌ | ❌ | Размер словаря |
| Bag of Words | ❌ | ❌ | Размер словаря |
| N-граммы | Частично | ❌ | Очень большой |
| TF-IDF | ❌ | ❌ | Размер словаря |

# Типичные ошибки

❌ One-Hot Encoding понимает смысл слов.

❌ Bag of Words учитывает порядок слов.

❌ TF-IDF использует нейронную сеть.

❌ N-граммы полностью решают проблему порядка слов.

❌ Эти методы больше нигде не применяются.

---

# Что спрашивают чаще всего?

- Почему текст нужно переводить в числа?
- Что такое One-Hot Encoding?
- Что такое разреженный вектор?
- Почему Bag of Words называется «мешком слов»?
- Какие недостатки есть у Bag of Words?
- Что такое N-граммы?
- Чем TF-IDF отличается от Bag of Words?
- Почему классические методы были вытеснены Word Embeddings?

---

# Что нужно помнить

✅ Компьютер умеет работать только с числами.

✅ One-Hot Encoding создаёт разреженные векторы и не отражает смысл слов.

✅ Bag of Words учитывает только частоту слов.

✅ N-граммы позволяют частично учитывать порядок слов.

✅ TF-IDF уменьшает влияние слишком частых слов.

✅ Ни один из классических методов не понимает семантическую близость слов.

✅ Ограничения этих подходов привели к появлению Word Embeddings.

# 7.2 Word Embeddings

> *«Самым большим прорывом раннего NLP стало понимание простой идеи: слова, которые встречаются в похожем окружении, обычно имеют похожий смысл.»*

После изучения этого раздела вы сможете:

- понимать идею Word Embeddings;
- объяснять Distributional Hypothesis;
- различать CBOW и Skip-Gram;
- понимать преимущества FastText и GloVe;
- объяснять ограничения классических эмбеддингов;
- отвечать на популярные вопросы интервью.

---

# Почему появились Word Embeddings?

В предыдущем разделе мы рассмотрели One-Hot Encoding, Bag of Words и TF-IDF.

Все эти методы имели один общий недостаток — они рассматривали слова как независимые объекты.

Для модели слова

```
кот
```

и

```
собака
```

были настолько же различны, как

```
кот
```

и

```
самолёт
```

Очевидно, это не соответствует реальности.

Хотелось получить такое представление слов, при котором похожие по смыслу слова автоматически оказывались бы рядом.

Именно эту задачу решили Word Embeddings.

---

# Distributional Hypothesis

## 📖 Что это?

Практически вся современная обработка текста основана на **Distributional Hypothesis**.

Она формулируется очень просто:

> **Слова, встречающиеся в похожем контексте, обычно имеют похожее значение.**

---

## Интуитивный пример

Представьте, что вы никогда раньше не слышали слово

```
кот
```

Но постоянно встречаете предложения:

```
Кот пьёт молоко.

Кот ловит мышей.

Кот спит на диване.

Кот мяукает.
```

Даже не зная определения слова, можно сделать вывод, что речь идёт о каком-то домашнем животном.

Именно такой логикой руководствуется Word2Vec.

---

> 🎯 Главное

Модель изучает не сами слова, а их окружение.

---

# Что такое Embedding?

## 📖 Что это?

**Embedding** — это плотный числовой вектор, описывающий объект.

В NLP объектом обычно является слово или токен.

Например,

```
кот
```

может быть представлен вектором

```
[0.31, -0.74, 1.28, ..., 0.12]
```

Такой вектор обычно имеет размерность от нескольких десятков до нескольких тысяч элементов.

Главное отличие от One-Hot Encoding заключается в том, что значения этого вектора обучаются автоматически.

---

# Почему Embeddings лучше One-Hot?

Рассмотрим основные отличия.

| One-Hot Encoding | Embedding |
|------------------|-----------|
| Размер = словарь | Размер фиксированный |
| Разреженный | Плотный |
| Не отражает смысл | Кодирует семантическую близость |
| Не обучается | Обучается вместе с моделью |

---

Самое важное свойство Embeddings заключается в том, что расстояние между словами начинает отражать их смысл.

Например,

```
кот
```

оказывается значительно ближе к

```
кошка
```

чем к

```
самолёт
```

---

# Word2Vec

## 📖 Что это?

**Word2Vec** — один из первых алгоритмов обучения качественных эмбеддингов слов.

Он был опубликован исследователями Google в 2013 году и быстро стал стандартом в NLP.

---

## Главная идея

Вместо того чтобы вручную задавать векторы слов, модель обучается решать простую задачу прогнозирования.

Если она научится хорошо предсказывать слова по контексту, значит её внутренние представления тоже будут содержать информацию о смысле слов.

---

# Две стратегии обучения

Word2Vec существует в двух вариантах:

- CBOW (Continuous Bag of Words);
- Skip-Gram.

---

# CBOW

## 📖 Что это?

CBOW пытается предсказать пропущенное слово, используя окружающий контекст.

Например,

```
Я люблю ____ обучение.
```

Используя слова

```
Я

люблю

обучение
```

модель должна восстановить слово

```
машинное
```

---

## Когда используется?

CBOW быстрее обучается и хорошо работает для частых слов.

---

# Skip-Gram

## 📖 Что это?

Skip-Gram решает противоположную задачу.

Модель получает центральное слово и должна предсказать слова вокруг него.

Например,

известно слово

```
машинное
```

Необходимо предсказать

```
люблю

обучение
```

---

## Когда используется?

Skip-Gram обычно лучше изучает редкие слова, но требует больше вычислений.

---

# Сравнение CBOW и Skip-Gram

| Характеристика | CBOW | Skip-Gram |
|----------------|------|-----------|
| Предсказывает | Центральное слово | Контекст |
| Скорость обучения | Выше | Ниже |
| Работа с редкими словами | Хуже | Лучше |

---

### 🎤 Вопрос с интервью

**Чем CBOW отличается от Skip-Gram?**

CBOW восстанавливает центральное слово по контексту, а Skip-Gram, наоборот, предсказывает контекст по центральному слову.

---

# Negative Sampling

Во время обучения возникает проблема.

Представьте словарь размером миллион слов.

Для каждого примера сравнивать слово со всеми остальными словами слишком дорого.

---

## Идея Negative Sampling

Вместо полного словаря модель сравнивает правильный ответ лишь с небольшим количеством случайно выбранных слов.

Например,

если правильное слово —

```
кошка
```

то модель дополнительно сравнивает его с несколькими случайными словами:

```
самолёт

яблоко

телефон

река
```

Такие слова называют **отрицательными примерами (negative samples)**.

Это значительно ускоряет обучение без существенной потери качества.

---

### Production Notes

Именно Negative Sampling сделал обучение Word2Vec достаточно быстрым для работы с очень большими корпусами текста.

---

# Самое известное свойство Word2Vec

После обучения оказывается, что эмбеддинги начинают отражать сложные семантические отношения.

Например,

```
Король
− Мужчина
+ Женщина
≈ Королева
```

или

```
Париж
− Франция
+ Италия
≈ Рим
```

Никто специально не обучал модель таким аналогиям.

Они появились автоматически благодаря структуре эмбеддингов.

---

### ⚠️ Важно

Не стоит воспринимать такие примеры как обязательное свойство любых эмбеддингов.

Это красивый эффект, который хорошо демонстрирует, что векторы действительно начинают хранить смысловую информацию.

---

# FastText

## 📖 Что это?

**FastText** — развитие Word2Vec, предложенное Meta.

---

## Главная идея

Word2Vec рассматривает слово как единое целое.

FastText разбивает слово на небольшие символьные фрагменты (n-граммы символов) и строит эмбеддинг на их основе.

---

## Почему это полезно?

Предположим, модель никогда раньше не встречала слово

```
биоинформатик
```

Но она знает части слова:

```
био

информ

матик
```

Благодаря этому FastText способен построить разумное представление даже для неизвестных слов.

---

### Где FastText особенно полезен?

FastText хорошо работает для языков с богатой морфологией:

- русский;
- немецкий;
- финский;
- турецкий.

---

### 🎤 Вопрос с интервью

**Почему FastText лучше Word2Vec для русского языка?**

Потому что он учитывает внутреннюю структуру слов и лучше работает с различными словоформами.

---

# GloVe

## 📖 Что это?

**GloVe (Global Vectors)** — ещё один алгоритм обучения эмбеддингов.

Он был разработан исследователями Stanford.

---

## Чем отличается от Word2Vec?

Word2Vec изучает локальный контекст.

GloVe использует глобальную статистику совместной встречаемости слов во всём корпусе.

Идея немного отличается, но конечная цель та же — получить качественные эмбеддинги.

---

### Production Notes

Исторически GloVe и Word2Vec показывали очень близкое качество.

Сегодня оба алгоритма в основном уступили место контекстным моделям.

---

# Ограничения классических Embeddings

Несмотря на огромный успех, Word2Vec, FastText и GloVe имеют фундаментальное ограничение.

Каждое слово получает **один единственный вектор**.

Например, слово

```
банк
```

будет иметь одинаковое представление в предложениях

> «Я положил деньги в банк.»

и

> «Мы сидели на берегу банка реки.»

Хотя смысл слова совершенно разный.

Такие эмбеддинги называют **контекстно-независимыми (static embeddings)**.

---

# Что появилось дальше?

Хотелось получить представление, которое зависело бы от конкретного предложения.

То есть слово

```
банк
```

в разных контекстах должно иметь разные эмбеддинги.

Именно эта идея привела к появлению **контекстных эмбеддингов**, а затем — моделей BERT и Transformer.

---

# Сравнение методов

| Метод | Главная идея | Ограничение |
|--------|--------------|-------------|
| Word2Vec | Изучает локальный контекст | Один вектор на слово |
| FastText | Использует части слов | Один вектор на слово |
| GloVe | Использует глобальную статистику | Один вектор на слово |

---

# Типичные ошибки

❌ Embedding и One-Hot Encoding — это одно и то же.

❌ Word2Vec понимает смысл предложений.

❌ FastText основан на Transformer.

❌ GloVe всегда лучше Word2Vec.

❌ Word2Vec строит разные эмбеддинги для одного и того же слова в разных предложениях.

✅ Word Embeddings представляют слова в виде плотных обучаемых векторов.

# 7.3 Последовательные модели

> *«Word2Vec научился понимать смысл отдельных слов. Следующая задача оказалась намного сложнее — научить модель понимать смысл целого предложения.»*

- понимать, почему Word Embeddings недостаточно для обработки текста;
- объяснять принцип работы RNN;

- различать RNN, LSTM и GRU;
- понимать архитектуру Seq2Seq;

---

# Почему одних Embeddings недостаточно?

Word2Vec умеет строить хорошие представления отдельных слов.

Но предложения состоят не из набора независимых слов, а из последовательностей.

Например,

> **«Я не люблю машинное обучение.»**

и

> **«Я люблю машинное обучение.»**

отличаются всего одним словом, но имеют противоположный смысл.

Кроме того, смысл слова часто зависит от того, какие слова стоят рядом.

Следовательно, модели необходимо учитывать **контекст** и **порядок слов**.

---

# Почему MLP плохо подходит для текста?

Представим предложение

```
Я люблю машинное обучение.
```

Если подать эмбеддинги слов в обычный MLP, модель увидит лишь фиксированный набор чисел.

Она не знает:

- какое слово было первым;
- какое — последним;
- какие слова находятся рядом;
- что предложение является последовательностью.

Поэтому обычные полносвязные сети плохо подходят для обработки текста.

---

# Что хотелось получить?

Нужна была модель, которая:

- читает слова по порядку;
- постепенно накапливает информацию;
- помнит предыдущие слова;
- использует эту информацию при обработке следующих слов.

Так появилась **Recurrent Neural Network**.

---

# RNN

## 📖 Что это?

**Recurrent Neural Network (RNN)** — нейронная сеть, предназначенная для обработки последовательностей.

Главная идея RNN заключается в том, что модель читает элементы один за другим и после каждого шага сохраняет некоторую информацию о прочитанном.

---

# Hidden State

## 📖 Что это?

**Hidden State** — внутреннее состояние (или память) RNN.

После обработки каждого нового слова модель обновляет Hidden State.

Именно он содержит информацию обо всём, что было прочитано ранее.

---

## Интуиция

Представьте, что вы читаете книгу.

После каждой страницы вы не забываете предыдущие события, а продолжаете держать их в памяти.

Hidden State выполняет ту же роль.

Он постоянно обновляется по мере чтения текста.

---

# Как работает RNN?

Пусть модель получает предложение

```
Я люблю NLP.
```

Процесс выглядит следующим образом.

```text
Я
 │
 ▼
Hidden State₁
 │
 ▼
люблю
 │
 ▼
Hidden State₂
 │
 ▼
NLP
 │
 ▼
Hidden State₃
```

После обработки каждого слова модель обновляет своё внутреннее состояние.

Таким образом, информация постепенно переносится вдоль всей последовательности.

---

> 🎯 Главное

RNN запоминает не весь текст целиком, а некоторое внутреннее представление этого текста.

---

# Где использовались RNN?

До появления Transformer RNN были основной архитектурой для большинства NLP-задач.

Например:

- машинный перевод;
- генерация текста;
- анализ тональности;
- распознавание речи;
- языковое моделирование;
- обработка временных рядов.

---

# Обучение RNN

Для обучения RNN используется специальная версия алгоритма Backpropagation.

Она называется **Backpropagation Through Time (BPTT)**.

---

## Нужно ли знать детали?

Нет.

Для большинства интервью достаточно понимать следующую идею.

Обычный Backpropagation распространяет ошибку через слои сети.

BPTT распространяет ошибку не только через слои, но и через временные шаги последовательности.

---

### 🎤 Вопрос с интервью

**Что такое BPTT?**

Это модификация Backpropagation для рекуррентных нейронных сетей, которая учитывает временную структуру последовательности.

---

# Главная проблема RNN

Представим длинное предложение.

> **«Книга, которую я купил несколько месяцев назад в небольшом магазине рядом с университетом, оказалась очень интересной.»**

Чтобы правильно интерпретировать слово

> **«интересной»**

необходимо помнить, что речь шла о книге, упомянутой в самом начале предложения.

Обычная RNN постепенно начинает терять информацию о первых словах.

Чем длиннее последовательность, тем сильнее становится эта проблема.

---

# Vanishing Gradient

Мы уже обсуждали этот термин в предыдущей главе.

В RNN проблема проявляется особенно сильно.

Во время обучения градиенты проходят через множество временных шагов.

Со временем они становятся всё меньше, и модель практически перестаёт учитывать далёкий контекст.

Именно поэтому обычные RNN плохо работают с длинными последовательностями.

---

# LSTM

## 📖 Что это?

**Long Short-Term Memory (LSTM)** — улучшенная версия RNN, специально созданная для работы с длинными последовательностями.

---

## Главная идея

Вместо одной простой памяти LSTM использует более сложный механизм управления информацией.

Модель самостоятельно решает:

- что сохранить;
- что забыть;
- какую информацию использовать дальше.

Благодаря этому LSTM значительно лучше запоминает долгосрочные зависимости.

---

# Gates в LSTM

Хотя внутреннее устройство LSTM достаточно сложное, на интервью обычно достаточно понимать назначение трёх основных механизмов.

| Gate | Назначение |
|------|------------|
| Forget Gate | Удаляет ненужную информацию |
| Input Gate | Добавляет новую информацию |
| Output Gate | Передаёт информацию следующему шагу |

---

### Нужно ли помнить формулы?

Нет.

Важно понимать только общую идею: LSTM умеет управлять своей памятью значительно лучше обычной RNN.

---

### 🎤 Вопрос с интервью

**Почему LSTM работает лучше обычной RNN?**

Потому что она может контролировать, какую информацию сохранить, а какую забыть, благодаря специальным механизмам управления памятью.

---

# GRU

## 📖 Что это?

**Gated Recurrent Unit (GRU)** — ещё одна разновидность рекуррентной сети.

Её можно рассматривать как упрощённую версию LSTM.

---

## Главная идея

GRU сохраняет большинство преимуществ LSTM, но использует более простую архитектуру.

За счёт этого:

- содержит меньше параметров;
- быстрее обучается;
- требует меньше памяти.

Во многих задачах качество GRU оказывается сопоставимым с LSTM.

---

### 🎤 Вопрос с интервью

**Чем GRU отличается от LSTM?**

GRU проще по архитектуре, содержит меньше параметров и обычно обучается быстрее, сохраняя сопоставимое качество.

---

# Bidirectional RNN

До сих пор мы рассматривали модель, читающую текст только слева направо.

Однако иногда для понимания слова важно знать и последующий контекст.

Например,

> **«Я открыл банк...»**

Пока предложение не закончится, невозможно понять, идёт ли речь о финансовой организации или берегу реки.

Для таких случаев используются **Bidirectional RNN**, которые читают последовательность одновременно в обоих направлениях.

---

# Seq2Seq

## 📖 Что это?

**Sequence-to-Sequence (Seq2Seq)** — архитектура, преобразующая одну последовательность в другую.

Наиболее известный пример — машинный перевод.

```text
Русское предложение
        │
        ▼
     Encoder
        │
        ▼
Внутреннее представление
        │
        ▼
     Decoder
        │
        ▼
English sentence
```

---

# Из чего состоит Seq2Seq?

Архитектура включает две основные части.

## Encoder

Читает входную последовательность и преобразует её во внутреннее представление.

---

## Decoder

Использует это представление для генерации новой последовательности.

---

# Главная проблема Seq2Seq

Encoder должен представить всё предложение одним вектором.

Если предложение короткое, этого обычно достаточно.

Но при увеличении длины текста информация начинает теряться.

Получается своеобразное «узкое место» (bottleneck).

Именно оно стало главным ограничением Seq2Seq.

---

> 🎯 Главное

Чем длиннее предложение, тем сложнее Encoder сохранить всю необходимую информацию в одном векторе.

---

# Почему последовательные модели были вытеснены?

Несмотря на успех, RNN, LSTM и GRU обладали рядом серьёзных ограничений.

## 1. Последовательные вычисления

Следующее слово невозможно обработать, пока не обработано предыдущее.

Из-за этого модель плохо масштабируется на GPU.

---

## 2. Проблемы с длинным контекстом

Даже LSTM постепенно теряет информацию при очень длинных последовательностях.

---

## 3. Низкая скорость обучения

Последовательная обработка делает обучение значительно медленнее по сравнению с Transformer.

---

# Сравнение моделей

| Архитектура | Преимущество | Недостаток |
|-------------|--------------|------------|
| RNN | Простая архитектура | Плохо работает с длинным контекстом |
| LSTM | Лучше запоминает информацию | Более сложная и медленная |
| GRU | Проще LSTM | Иногда немного уступает по качеству |
| Seq2Seq | Генерация последовательностей | Узкое место в виде одного вектора |

---

### 🚀 Production Notes

Сегодня RNN и LSTM значительно реже используются в NLP, однако они по-прежнему встречаются:

- при обработке временных рядов;
- в некоторых системах распознавания речи;
- в embedded-устройствах;
- в legacy-проектах.

Понимание этих архитектур остаётся важным, потому что именно их ограничения привели к появлению Attention.

❌ Hidden State содержит все предыдущие слова.

❌ GRU всегда лучше LSTM.

❌ LSTM полностью решила проблему длинного контекста.

❌ Seq2Seq и Transformer — это одна и та же архитектура.

# 7.4 Attention

> *«Attention стал самой важной идеей современного NLP. Именно он решил проблему, которая годами ограничивала возможности RNN и LSTM.»*

---

# Цели раздела

После изучения этого раздела вы сможете:

- понимать, зачем появился механизм Attention;
- объяснять ограничения Seq2Seq;
- понимать роли Query, Key и Value;
- понимать принцип работы Attention;
- различать Attention и Self-Attention;
- понимать, почему именно Attention сделал возможным появление Transformer.

---

# Почему появился Attention?

В предыдущем разделе мы познакомились с архитектурой Seq2Seq.

Напомним её устройство.

```text
Входной текст
      │
      ▼
   Encoder
      │
      ▼
 Один вектор
      │
      ▼
   Decoder
      │
      ▼
Выходной текст
```

Главная проблема скрывается именно здесь.

Encoder должен прочитать всё предложение и представить его одним вектором фиксированного размера.

Если предложение содержит десять слов — обычно всё работает хорошо.

Если предложение содержит несколько сотен слов — часть информации неизбежно теряется.

---

# Почему одного вектора недостаточно?

Представьте, что вы прочитали книгу объёмом 500 страниц.

После этого вам разрешили записать её содержание только на одной небольшой карточке.

Теперь другой человек должен восстановить всю книгу, используя только эту карточку.

Очевидно, сделать это практически невозможно.

Именно в такой ситуации находился Decoder в классическом Seq2Seq.

---

# Какая идея лежит в основе Attention?

Вместо того чтобы передавать Decoder только один итоговый вектор, можно предоставить ему доступ ко всем промежуточным представлениям Encoder.

Тогда при генерации каждого нового слова модель сможет самостоятельно выбирать, какая часть исходного предложения сейчас наиболее важна.

Именно эта идея получила название **Attention**.

---

> 🎯 Главное

Attention позволяет модели обращаться не ко всему предложению сразу, а только к наиболее важным его частям.

---

# Интуитивный пример

Рассмотрим предложение

> **«Я купил новую красную машину.»**

Предположим, модель сейчас генерирует слово

> **«машину»**.

Очевидно, в этот момент ей гораздо важнее обратить внимание на слова

- купил;
- новую;
- красную,

чем, например, на местоимение

> **«я»**.

Attention позволяет сделать именно это.

---

# Query, Key и Value

Практически каждое интервью по NLP содержит вопрос про эти три понятия.

На самом деле они значительно проще, чем кажется.

---

## Query

**Query** — это запрос.

Он отвечает на вопрос:

> **«Какую информацию мне сейчас нужно найти?»**

---

## Key

**Key** — описание информации.

Каждое слово имеет некоторый "ключ", который позволяет понять, насколько оно подходит текущему запросу.

---

## Value

**Value** — сама информация, которую модель получит, если выберет соответствующее слово.

---

# Почему используются именно такие названия?

Представьте библиотеку.

Вы приходите к библиотекарю и говорите:

> «Мне нужна книга по Transformer.»

Это ваш **Query**.

Далее библиотекарь просматривает каталог книг.

Карточки каталога — это **Key**.

После того как нужная книга найдена, вы получаете её содержимое.

Это **Value**.

В результате процесс выглядит следующим образом.

```text
Query
   │
   ▼
Сравнение со всеми Key
   │
   ▼
Выбор наиболее подходящих
   │
   ▼
Получение соответствующих Value
```

---

### 🚀 Production Notes

Именно поэтому Attention иногда называют **дифференцируемым поиском (Differentiable Retrieval)**.

По сути модель постоянно выполняет поиск информации внутри собственного представления текста.

Позже мы увидим очень похожую идею в RAG, где поиск уже происходит не внутри модели, а по внешней базе знаний.

---

# Как работает Attention?

Процесс можно разбить на несколько простых шагов.

### Шаг 1

Формируется Query.

Именно он определяет, какую информацию необходимо найти.

---

### Шаг 2

Query сравнивается со всеми Key.

Для каждого слова вычисляется степень его важности.

---

### Шаг 3

Полученные оценки преобразуются в веса.

Чем выше вес, тем сильнее модель обращает внимание на соответствующее слово.

---

### Шаг 4

Итоговое представление строится как комбинация Value с учётом найденных весов.

Именно оно передаётся дальше по сети.

---

# Почему используется скалярное произведение?

Очень популярный вопрос на собеседованиях.

Нужно понять, насколько Query похож на каждый Key.

Для этого используется **скалярное произведение**.

Интуитивно всё довольно просто.

Если два вектора направлены примерно одинаково, их скалярное произведение будет большим.

Если направления сильно отличаются — значение окажется небольшим.

Следовательно,

- большое значение означает: **«это слово сейчас важно»**;
- маленькое значение означает: **«на него можно почти не обращать внимания»**.

Важно понимать, что модель сама обучается строить такие Query и Key, при которых нужные слова становятся максимально похожими.

---

# Почему используется Softmax?

После вычисления оценок Attention получает набор произвольных чисел.

Например,

```
4.8
2.1
0.7
-1.3
```

Работать с ними неудобно.

Поэтому применяется функция **Softmax**.

Она превращает оценки в вероятности.

Например,

```
0.72
0.19
0.07
0.02
```

Теперь сразу видно, каким словам модель уделяет основное внимание.

---

# Почему Attention называют дифференцируемым поиском?

Обычный поиск работает дискретно.

Например,

```
Найти документ №17.
```

Получаем только один документ.

Attention работает иначе.

Он говорит:

- первое слово важно на 60%;
- второе — на 25%;
- третье — на 10%;
- остальные почти не влияют.

Затем модель объединяет информацию сразу от нескольких слов.

Именно поэтому механизм можно обучать обычным Backpropagation.

---

# Self-Attention

До сих пор мы рассматривали механизм Attention между Encoder и Decoder.

Но существует ещё более важная разновидность.

---

## 📖 Что это?

**Self-Attention** позволяет словам взаимодействовать друг с другом внутри одной и той же последовательности.

Другими словами, каждое слово может анализировать все остальные слова предложения.

---

## Интуитивный пример

Предложение

> **«Кошка сидит на ковре.»**

При обработке слова

> **«сидит»**

модель может обратить внимание прежде всего на слова

- кошка;
- ковре.

При этом предлог

> **«на»**

окажется значительно менее важным.

---

# Чем Self-Attention отличается от обычного Attention?

Обычный Attention связывает две разные последовательности.

Например,

Encoder и Decoder.

Self-Attention работает внутри одной последовательности.

Именно он является главным компонентом Transformer.

---

# Что изучают разные головы Attention?

Transformer использует сразу несколько механизмов Attention.

Возникает естественный вопрос.

Неужели все они делают одно и то же?

На практике — нет.

Исследования OpenAI, Google и Anthropic показывают, что разные головы часто специализируются.

Например,

- одна отслеживает синтаксические связи;
- другая связывает местоимения с существительными;
- третья ищет длинные зависимости;
- четвёртая концентрируется на знаках препинания.

Важно понимать, что никто не программирует эти роли вручную.

Во время обучения каждая голова самостоятельно находит наиболее полезную стратегию.

---

# Почему Attention оказался лучше RNN?

У RNN информация должна пройти через всю последовательность.

Если между двумя словами находится 300 других слов, сигналу приходится пройти через все эти шаги.

Attention работает иначе.

Любое слово может обратиться к любому другому напрямую.

Расстояние между ними практически перестаёт играть роль.

Именно благодаря этому Transformer значительно лучше работает с длинными зависимостями.

---

# Недостатки Attention

Несмотря на все преимущества, механизм Attention не идеален.

Главная проблема заключается в вычислительной сложности.

Если последовательность содержит **N** токенов, необходимо сравнить каждый токен со всеми остальными.

Количество сравнений становится пропорционально

```
N²
```

---

# Почему это проблема?

Предположим, длина контекста увеличилась в 10 раз.

Тогда число операций увеличится примерно в 100 раз.

Именно поэтому обработка очень длинных документов требует огромных вычислительных ресурсов.

Сегодня эта проблема активно решается с помощью различных модификаций Attention.

Например:

- FlashAttention;
- Sparse Attention;
- Linear Attention;
- Longformer;
- Performer.

Подробно мы познакомимся с ними в главе про LLM.

---

# Сравнение подходов

| Архитектура | Главное ограничение |
|--------------|--------------------|
| RNN | Теряет длинный контекст |
| LSTM | Последовательные вычисления |
| Seq2Seq | Один вектор для всего предложения |
| Attention | Квадратичная вычислительная сложность |

---

### 🚀 Production Notes

Сегодня механизм Attention используется далеко не только в NLP.

Он активно применяется:

- в Computer Vision (Vision Transformer);
- в мультимодальных моделях;
- в рекомендательных системах;
- в моделях обработки белков;
- практически во всех современных LLM.

Attention давно перестал быть исключительно NLP-технологией.

---

❌ Attention и Self-Attention — одно и то же.

❌ Query, Key и Value — это входные данные модели.

❌ Attention выбирает только одно слово.

❌ Attention полностью заменяет нейронную сеть.

❌ Все головы Attention изучают одинаковые зависимости.

✅ Query — запрос, Key — описание информации, Value — сама информация.

✅ Скалярное произведение используется для оценки похожести Query и Key.

✅ Softmax превращает оценки в веса внимания.

✅ Self-Attention позволяет словам взаимодействовать друг с другом внутри одной последовательности.

✅ Разные головы Attention изучают разные типы зависимостей.

✅ Главный недостаток Attention — квадратичная сложность по длине последовательности.

# 7.5 Transformer

> *«Transformer не просто заменил RNN. Он полностью изменил способ, которым нейронные сети работают с последовательностями.»*

---

# Цели раздела

После изучения этого раздела вы сможете:

- понимать архитектуру Transformer;
- объяснять назначение каждого блока;
- понимать различия между Encoder и Decoder;
- понимать устройство современных LLM;
- отвечать практически на любые вопросы интервью по Transformer.

---

# История появления

В 2017 году исследователи Google опубликовали статью

> **Attention Is All You Need**

Именно в ней впервые была представлена архитектура Transformer.

До этого почти все модели NLP были построены на RNN и LSTM.

После публикации статьи направление развития всей области изменилось.

Сегодня практически все современные большие языковые модели являются различными вариантами Transformer.

---

# Главная идея Transformer

RNN читает текст последовательно.

```
Слово 1

↓

Слово 2

↓

Слово 3

↓

...
```

Transformer делает иначе.

Он получает всю последовательность сразу.

Каждое слово может взаимодействовать с любым другим словом независимо от расстояния между ними.

Именно поэтому Transformer значительно лучше масштабируется на современных GPU.

---

# Общая архитектура

Классический Transformer выглядит следующим образом.

```text
Входной текст
      │
      ▼
 Embedding
      │
      ▼
Positional Encoding
      │
      ▼
 Encoder × N
      │
      ▼
 Decoder × N
      │
      ▼
 Linear
      │
      ▼
 Softmax
```

Не стоит пугаться количества блоков.

Практически все они нам уже знакомы.

---

# Почему сначала Embedding?

Компьютер по-прежнему не умеет работать со словами.

Поэтому первым этапом каждое слово преобразуется в Embedding.

После этого вся модель работает исключительно с векторами.

---

# Почему нужен Positional Encoding?

Self-Attention рассматривает все слова одновременно.

Поэтому сам по себе Transformer не знает порядок слов.

Например,

> «Собака укусила человека.»

и

> «Человек укусил собаку.»

содержат одинаковый набор слов.

Без дополнительной информации Transformer не сможет различить эти предложения.

Поэтому к каждому Embedding добавляется информация о позиции слова.

---

> 🎯 Главное

Transformer не умеет понимать порядок слов самостоятельно.

Информация о позиции должна быть добавлена явно.

---

# Encoder

Encoder состоит из нескольких одинаковых блоков.

Каждый блок включает:

```text
Multi-Head Attention
        │
        ▼
Add & LayerNorm
        │
        ▼
Feed Forward Network
        │
        ▼
Add & LayerNorm
```

Затем этот блок повторяется несколько раз.

---

# Почему блок повторяется?

Возникает естественный вопрос.

Почему нельзя использовать один очень большой Encoder?

Ответ такой же, как и в CNN.

Первые слои изучают простые зависимости.

Последующие — более сложные.

Каждый новый Encoder Block постепенно уточняет представление текста.

---

# Self-Attention

Self-Attention — главный компонент Transformer.

Именно здесь каждое слово получает информацию обо всех остальных словах предложения.

Например,

при обработке слова

> **«сидит»**

модель может обратить внимание на

- кошка;
- ковре;

и практически проигнорировать предлог

> **«на»**.

---

# Почему после Attention стоит Feed Forward Network?

Это один из самых популярных вопросов на хороших интервью.

Self-Attention отвечает только за обмен информацией между словами.

После этого необходимо отдельно обработать представление каждого слова.

Именно этим занимается Feed Forward Network (FFN).

Можно сказать,

что Attention позволяет словам "пообщаться",

а FFN помогает каждому слову самостоятельно обработать полученную информацию.

---

### Production Notes

Практически во всех современных Transformer FFN содержит значительно больше параметров, чем механизм Attention.

Именно поэтому оптимизация FFN является важной частью ускорения современных LLM.

---

# Multi-Head Attention

Возникает ещё один вопрос.

Почему Attention несколько?

Почему не использовать только один?

Каждая голова внимания может изучать разные закономерности.

Например,

- синтаксис;
- грамматические связи;
- длинные зависимости;
- местоимения;
- пунктуацию.

После этого результаты всех голов объединяются.

Именно поэтому используется термин **Multi-Head Attention**.

---

# Почему несколько Attention лучше одного?

Представьте консилиум врачей.

Каждый специалист смотрит на пациента со своей точки зрения.

После этого принимается общее решение.

Multi-Head Attention работает очень похоже.

Разные головы специализируются на разных аспектах текста.

---

# Residual Connection

После каждого большого блока используется Residual Connection.

Мы уже обсуждали её в предыдущей главе.

Главная задача — облегчить обучение очень глубоких моделей.

Без Residual Connection обучение Transformer с десятками и сотнями слоёв было бы значительно сложнее.

---

# LayerNorm

После каждого Residual Connection применяется LayerNorm.

Её задача — сделать обучение стабильнее.

Сегодня практически все Transformer используют именно LayerNorm.

---

# Почему сначала Add, а потом LayerNorm?

В классическом Transformer порядок был именно таким.

Такой вариант называют **Post-LayerNorm**.

Позже оказалось, что для очень глубоких моделей стабильнее сначала выполнять LayerNorm, а затем остальные вычисления.

Так появился **Pre-LayerNorm**.

---

### Нужно ли помнить это на интервью?

Для Junior — нет.

Для Middle и Senior желательно понимать, что современные GPT-подобные модели чаще используют именно **Pre-LayerNorm**.

---

# Decoder

До сих пор мы говорили только про Encoder.

Теперь рассмотрим Decoder.

Он похож на Encoder, но содержит ещё один дополнительный механизм внимания.

Кроме того, Decoder использует **Masked Self-Attention**.

---

# Почему Decoder использует Masked Self-Attention?

Представьте,

что модель должна предсказать следующее слово.

Она не должна видеть будущее.

Например,

если необходимо предсказать слово

> **«обучение»**

в предложении

> «Я люблю машинное обучение.»

модель не должна заранее знать слово

> **«обучение»**.

Поэтому Decoder скрывает все будущие токены.

Это называется **Masked Self-Attention**.

---

### 🎤 Вопрос с интервью

**Почему GPT использует Masked Self-Attention?**

Потому что GPT генерирует текст авторегрессионно и не должен видеть будущие токены во время обучения.

---

# Encoder-only модели

Не каждой задаче нужен Decoder.

Например,

BERT использует только Encoder.

Такие модели хорошо подходят для:

- классификации;
- поиска;
- NER;
- анализа тональности;
- построения эмбеддингов.

Главная задача Encoder — хорошо понимать входной текст.

---

# Decoder-only модели

GPT,

Llama,

Qwen,

Gemma,

DeepSeek

используют только Decoder.

---

# Почему GPT отказался от Encoder?

Очень популярный вопрос.

GPT решает задачу генерации следующего токена.

Для этого достаточно Decoder.

Encoder нужен тогда, когда есть отдельная входная последовательность, например при машинном переводе.

Для генерации текста он просто не требуется.

---

# Encoder-Decoder модели

Некоторые модели используют оба блока.

Например,

T5.

Такие архитектуры особенно удобны для задач преобразования одной последовательности в другую.

Например,

- машинный перевод;
- суммаризация;
- перефразирование.

---

# Почему BERT нельзя использовать как GPT?

Это ещё один классический вопрос интервью.

BERT обучается видеть текст сразу целиком.

Во время обучения ему доступны и предыдущие, и последующие слова.

GPT обучается иначе.

Он всегда предсказывает только следующий токен.

Поэтому GPT естественным образом подходит для генерации текста.

BERT — нет.

---

# Почему Transformer оказался лучше RNN?

Transformer решил сразу несколько фундаментальных проблем.

**Во-первых**, все слова обрабатываются одновременно.

Это позволяет эффективно использовать GPU.

**Во-вторых**, модель лучше работает с длинными зависимостями.

Любое слово может сразу обратиться к любому другому слову.

**В-третьих**, архитектура хорошо масштабируется.

Именно поэтому современные модели содержат десятки и сотни слоёв.

---

# Главный недостаток Transformer

Несмотря на все преимущества, у Transformer есть серьёзное ограничение.

Self-Attention требует сравнить каждый токен со всеми остальными.

Если длина последовательности равна **N**, количество сравнений растёт пропорционально **N²**.

Именно поэтому длинный контекст остаётся дорогим по памяти и времени вычислений.

Эта проблема активно решается в современных исследованиях.

---

# Сравнение архитектур

| Архитектура | Лучше всего подходит для |
|------------|--------------------------|
| Encoder-only | Понимание текста |
| Decoder-only | Генерация текста |
| Encoder-Decoder | Преобразование последовательностей |

---

### 🚀 Production Notes

Практически все современные модели являются модификациями Transformer.

Меняются детали архитектуры:

- тип Position Encoding;
- вид нормализации;
- устройство FFN;
- разновидность Attention;

но базовая идея остаётся неизменной.

Именно поэтому понимание Transformer — обязательный навык для любого NLP Engineer.

---

# Типичные ошибки

❌ GPT состоит из Encoder и Decoder.

❌ BERT умеет генерировать текст так же, как GPT.

❌ Multi-Head Attention означает несколько независимых Transformer.

❌ FFN отвечает за взаимодействие между словами.

❌ Positional Encoding нужен только для длинных предложений.

# 7.6 Современные NLP-модели

> *«Transformer — это архитектура. BERT, GPT, T5 и другие модели — это разные способы её использования для решения различных задач.»*

- знать, какие задачи решает каждая архитектура;
- понимать, почему появились новые модификации BERT;
- различать Encoder-only, Decoder-only и Encoder-Decoder модели;
- отвечать на популярные вопросы интервью.

---

# После появления Transformer

После публикации статьи **Attention Is All You Need** стало понятно, что архитектура Transformer значительно превосходит RNN и LSTM.

Однако сама архитектура ещё не отвечала на вопрос:

> **Как именно её лучше обучать?**

В течение следующих нескольких лет появились модели, каждая из которых предлагала своё решение этой задачи.

Большинство современных архитектур можно разделить на три больших семейства.

| Тип модели | Представители | Основная задача |
|------------|---------------|-----------------|
| Encoder-only | BERT, RoBERTa, ALBERT | Понимание текста |
| Decoder-only | GPT, Llama, Qwen, Gemma | Генерация текста |
| Encoder-Decoder | T5 | Преобразование текста |

---

# BERT

## 📖 Что это?

**BERT (Bidirectional Encoder Representations from Transformers)** — одна из самых влиятельных моделей в истории NLP.

Она была представлена Google в 2018 году.

Главная идея BERT — научиться максимально хорошо понимать текст.

---

## Почему BERT называется Bidirectional?

Во время обучения модель одновременно видит слова слева и справа от текущего слова.

Например,

```
Я положил деньги в банк.
```

и

```
Мы сидели на берегу банка.
```

BERT использует весь контекст предложения и поэтому понимает, что слово **«банк»** в этих примерах имеет разный смысл.

Именно поэтому BERT строит **контекстные эмбеддинги**.

---

## Где используется?

BERT отлично подходит для задач, где необходимо понять текст.

Например:

- классификация документов;
- анализ тональности;
- поиск;
- Question Answering;
- Named Entity Recognition;
- получение эмбеддингов.

---

## Главный недостаток

BERT плохо подходит для генерации текста.

Причина очень проста.

Во время обучения модель всегда видела предложение целиком.

Во время генерации будущих слов ещё не существует.

Следовательно, процесс обучения и генерации не совпадают.

---

### 🎤 Вопрос с интервью

**Почему BERT нельзя использовать как ChatGPT?**

Потому что BERT обучается видеть весь текст одновременно, а генерация требует предсказывать следующий токен, не зная будущих слов.

---

# GPT

## 📖 Что это?

**GPT (Generative Pre-trained Transformer)** — семейство моделей, разработанное OpenAI.

Главная задача GPT — генерация следующего токена.

---

## Главная идея

Во время обучения модель получает последовательность

```
Я люблю машинное
```

и должна предсказать

```
обучение
```

Затем

```
Я люблю машинное обучение
```

↓

предсказать следующее слово.

И так миллиарды раз.

---

## Почему GPT использует только Decoder?

Потому что задача генерации текста не требует Encoder.

Достаточно Decoder с Masked Self-Attention, который никогда не видит будущие токены.

Именно это делает GPT естественным выбором для генеративных моделей.

---

## Где используется?

Практически все современные LLM относятся к этому семейству.

Например:

- GPT;
- Llama;
- Qwen;
- Gemma;
- DeepSeek;
- Mistral.

---

### 🎤 Вопрос с интервью

**Почему GPT использует Decoder-only архитектуру?**

Потому что задача модели — авторегрессионная генерация следующего токена. Для неё достаточно Decoder, а использование Encoder только усложнило бы архитектуру без существенной пользы.

---

# BERT vs GPT

Это один из самых популярных вопросов на интервью.

| BERT | GPT |
|------|-----|
| Encoder-only | Decoder-only |
| Понимает текст | Генерирует текст |
| Видит весь контекст | Видит только прошлые токены |
| Хорош для классификации | Хорош для генерации |

---

> 🎯 Главное

BERT отвечает на вопрос:

> **«Что означает этот текст?»**

GPT отвечает на вопрос:

> **«Какой токен должен идти следующим?»**

---

# RoBERTa

## 📖 Что это?

**RoBERTa (Robustly Optimized BERT Approach)** — улучшенная версия BERT.

---

## Что изменилось?

Архитектура практически не изменилась.

Исследователи показали, что BERT можно значительно улучшить без изменения самой модели.

Основные изменения:

- больше данных;
- более длительное обучение;
- улучшенная стратегия обучения.

---

### Главное

RoBERTa — это не новая архитектура.

Это лучше обученный BERT.

---

# ALBERT

## 📖 Что это?

**ALBERT (A Lite BERT)** — облегчённая версия BERT.

---

## Зачем появился?

BERT оказался очень большим.

ALBERT уменьшает количество параметров с помощью нескольких инженерных решений.

При этом качество снижается совсем незначительно.

---

### Production Notes

ALBERT интересен прежде всего как пример эффективного уменьшения размера модели.

---

# DistilBERT

## 📖 Что это?

DistilBERT — компактная версия BERT.

Она обучается с помощью **Knowledge Distillation**.

---

## Что такое Distillation?

Большая модель (Teacher)

обучает

маленькую модель (Student).

Student старается повторить поведение Teacher.

В результате получается значительно более быстрая модель с небольшими потерями качества.

---

### Где используется?

Когда важны:

- скорость;
- небольшой размер модели;
- инференс на слабом оборудовании.

---

# ELECTRA

## 📖 Что это?

ELECTRA предложила другую идею предварительного обучения.

Вместо восстановления пропущенных слов модель должна определить,

какие слова были заменены генератором.

---

## Почему это интересно?

Во время обучения используется информация практически от каждого токена.

Это делает обучение значительно эффективнее по сравнению с классическим Masked Language Modeling.

---

### Нужно ли знать детали?

Нет.

Для большинства интервью достаточно понимать,

что ELECTRA предложила альтернативную стратегию предварительного обучения.

---

# T5

## 📖 Что это?

**T5 (Text-to-Text Transfer Transformer)** — модель Google, использующая полный Transformer (Encoder + Decoder).

---

## Главная идея

Любую задачу NLP можно представить как преобразование текста в текст.

Например,

```
Translate English to German:
Hello

↓

Hallo
```

или

```
Summarize:

<документ>

↓

<краткое содержание>
```

или

```
Question:

...

↓

Answer:

...
```

Все задачи имеют одинаковый интерфейс.

---

## Где используется?

T5 хорошо подходит для задач,

в которых необходимо преобразовать одну последовательность в другую.

---

# Какую модель выбрать?

Очень распространённый вопрос.

Если необходимо понять текст,

обычно выбирают Encoder-only модель.

Если нужно генерировать текст,

используют Decoder-only модель.

Если требуется преобразование одной последовательности в другую,

подходит Encoder-Decoder.

---

# Современное состояние NLP

Если посмотреть на развитие последних лет,

станет заметна интересная тенденция.

Большинство новых крупных моделей —

это Decoder-only архитектуры.

Причина проста.

Именно они лучше всего подходят для генерации текста,

которая стала центральной задачей современных LLM.

Однако Encoder-модели по-прежнему активно используются

для поиска,

классификации

и построения эмбеддингов.

---

### 🚀 Production Notes

Сегодня во многих production-системах используются сразу две модели.

Например:

- Encoder-модель строит эмбеддинги документов для поиска.
- Decoder-модель генерирует итоговый ответ пользователю.

Именно так устроены многие современные RAG-системы.

---

# Сравнение моделей

| Модель | Тип | Основная задача |
|---------|-----|-----------------|
| BERT | Encoder-only | Понимание текста |
| RoBERTa | Encoder-only | Улучшенный BERT |

❌ BERT и GPT отличаются только размером.

❌ GPT использует Encoder.

❌ T5 — это улучшенная версия GPT.

❌ RoBERTa имеет совершенно новую архитектуру.

❌ DistilBERT обучается с нуля.

---

- Чем BERT отличается от GPT?
- Почему GPT использует Decoder-only?
- Почему BERT плохо подходит для генерации?
- Что такое RoBERTa?
- Зачем появился ALBERT?
- Что такое Knowledge Distillation?
- Чем ELECTRA отличается от BERT?

- Какую архитектуру выбрать для конкретной задачи?

---

# Что нужно помнить

✅ BERT предназначен для понимания текста.

✅ GPT предназначен для генерации текста.

✅ RoBERTa — улучшенный способ обучения BERT.

✅ ALBERT уменьшает количество параметров.

✅ ELECTRA предлагает альтернативный способ предварительного обучения.

# 7.7 Production NLP

> *«На практике NLP начинается не с Transformer, а с Tokenizer. Если токенизация работает неправильно, модель никогда не увидит корректный вход.»*

# Почему нельзя просто подавать текст в модель?

Transformer работает только с числами.

Даже после появления Embedding проблема полностью не исчезла.

Сначала необходимо разбить текст на небольшие элементы, которые модель сможет обработать.

Именно этим занимается **Tokenizer**.

---

# Что такое Tokenizer?

## 📖 Что это?

**Tokenizer** — алгоритм, который преобразует текст в последовательность токенов.

После этого каждому токену сопоставляется числовой идентификатор (**Token ID**).

Например,

```
I love machine learning.
```

может превратиться в

```
["I", "love", "machine", "learning", "."]
```

а затем

```
[51, 842, 1907, 7341, 13]
```

Именно эта последовательность поступает в модель.

---

# Что такое токен?

Важно понимать,

что **токен — это не всегда слово**.

В зависимости от Tokenizer токеном может быть:

- слово;
- часть слова;
- отдельный символ;
- знак препинания;
- пробел.

Например,

```
unbelievable
```

может быть разбито как

```
["un", "believ", "able"]
```

или

```
["unbelievable"]
```

или даже

```
["u", "n", "b", ...]
```

---

### 🎤 Вопрос с интервью

**Токен и слово — это одно и то же?**

Нет.

Современные модели почти всегда работают с подсловными токенами (subword tokens).

---

# Почему не использовать слова?

На первый взгляд кажется,

что каждое слово можно сделать отдельным токеном.

Но возникает несколько проблем.

Во-первых,

словарь становится огромным.

Во-вторых,

появляются неизвестные слова.

Например,

```
биоинформатик
```

или

```
ChatGPT-7
```

Если их нет в словаре,

модель не сможет их обработать.

Поэтому современные Tokenizer используют **подсловные единицы**.

---

# Subword Tokenization

Главная идея очень проста.

Вместо хранения каждого слова целиком

хранить часто встречающиеся части слов.

Например,

```
невероятный
```

↓

```
не

вероят

ный
```

Из ограниченного количества таких частей можно собрать практически любое новое слово.

Именно поэтому современные модели умеют работать даже с незнакомыми словами.

---

# BPE (Byte Pair Encoding)

## 📖 Что это?

**Byte Pair Encoding (BPE)** — один из самых популярных алгоритмов токенизации.

Он используется, например, в GPT-2.

---

## Главная идея

Обучение начинается с отдельных символов.

Затем наиболее часто встречающиеся пары символов постепенно объединяются.

Например,

```
m

a

c

h

i

n

e
```

↓

```
ma

ch

ine
```

↓

```
machine
```

Чем чаще последовательность встречается в корпусе,

тем выше вероятность,

что она станет отдельным токеном.

---

### Production Notes

BPE позволяет ограничить размер словаря,

при этом сохранив возможность представлять практически любые слова.

---

# WordPiece

## 📖 Что это?

**WordPiece** — алгоритм токенизации,

используемый в BERT.

---

Главная идея похожа на BPE,

но критерий объединения частей слова отличается.

WordPiece выбирает объединения,

которые сильнее улучшают качество языковой модели.

На практике для большинства задач различия между BPE и WordPiece не принципиальны.

---

# SentencePiece

## 📖 Что это?

**SentencePiece** — ещё один популярный алгоритм токенизации.

Он используется,

например,

в T5,

Llama,

Gemma

и многих современных моделях.

---

## Чем отличается?

SentencePiece работает непосредственно с исходным текстом,

не требуя предварительного разбиения по словам.

Это делает его особенно удобным для языков,

в которых слова не отделяются пробелами,

например китайского или японского.

---

### Production Notes

Если вы работаете с современными open-source LLM,

велика вероятность,

что используется именно SentencePiece.

---

# Специальные токены

Помимо обычных слов существуют специальные токены,

которые помогают модели понимать структуру входных данных.

---

## [CLS]

Используется в BERT.

Добавляется в начало последовательности.

Его итоговое представление часто используется для классификации всего текста.

---

## [SEP]

Разделяет две последовательности.

Например,

```
Вопрос

[SEP]

Ответ
```

---

## [PAD]

Используется для дополнения коротких последовательностей.

Подробнее мы рассмотрим его немного позже.

---

## [MASK]

Используется только во время обучения BERT.

Именно его модель должна восстановить.

---

## BOS и EOS

Во многих генеративных моделях используются специальные токены:

- **BOS (Beginning Of Sequence)** — начало последовательности;
- **EOS (End Of Sequence)** — конец последовательности.

Они помогают модели понимать границы текста.

---

# Почему нужен Padding?

Transformer ожидает,

что все последовательности внутри одного батча имеют одинаковую длину.

Но предложения бывают разной длины.

Например,

```
Привет.
```

и

```
Сегодня прекрасная солнечная погода.
```

Имеют разное количество токенов.

Поэтому короткие последовательности дополняются специальным токеном

```
[PAD]
```

до длины самой длинной последовательности в батче.

---

# Truncation

Иногда предложение оказывается слишком длинным.

Например,

модель поддерживает максимум

```
4096
```

токенов,

а документ содержит

```
12000
```

В этом случае часть текста приходится удалить.

Этот процесс называется **Truncation**.

---

### 🎤 Вопрос с интервью

**Что произойдёт, если текст длиннее максимального контекста модели?**

Он будет обрезан, если не использовать специальные методы обработки длинного контекста.

---

# Attention Mask

После Padding возникает новая проблема.

Transformer не знает,

что токены

```
[PAD]
```

являются искусственными.

Если ничего не сделать,

модель начнёт обращать на них внимание.

---

## Решение

Создаётся **Attention Mask**.

Например,

```
Токены

[12, 95, 61, 0, 0]
```

Маска

```
[1, 1, 1, 0, 0]
```

Единица означает,

что токен настоящий.

Ноль —

что этот токен нужно игнорировать.

---

> 🎯 Главное

Attention Mask не позволяет модели учитывать искусственно добавленные токены.

---

# Полный путь текста до Transformer

Теперь можно собрать все этапы вместе.

```text
Текст
   │
   ▼
Tokenizer
   │
   ▼
Token IDs
   │
   ▼
Embedding
   │
   ▼
Positional Encoding
   │
   ▼
Transformer
   │
   ▼
Output
```

Именно этот процесс происходит практически во всех современных NLP-моделях.

---

# Что происходит во время инференса?

Когда пользователь отправляет запрос,

происходит следующая последовательность действий.

1. Текст разбивается на токены.
2. Каждый токен заменяется своим ID.
3. ID преобразуются в эмбеддинги.
4. Transformer вычисляет вероятности следующего токена.
5. Выбирается следующий токен.
6. Он добавляется к последовательности.
7. Процесс повторяется до генерации ответа.

Именно так работают ChatGPT, Claude, Gemini, Qwen, DeepSeek и большинство современных LLM.

---

### 🚀 Production Notes

На практике именно токенизация часто становится причиной неожиданных ошибок.

Например:

- разные модели используют разные Tokenizer;
- одинаковый текст может разбиваться на разное количество токенов;
- стоимость API обычно считается именно в токенах, а не в словах;
- длина контекста также измеряется в токенах.

Поэтому при работе с LLM важно всегда мыслить токенами, а не словами.

❌ Максимальная длина контекста измеряется в словах.

# Основные определения

### NLP

Область искусственного интеллекта, занимающаяся обработкой человеческого языка.

---

### Token

Минимальная единица текста, с которой работает модель.

Это не обязательно слово.

---

### Embedding

Плотное числовое представление слова или токена.

---

### Context

Окружающие слова, которые помогают определить смысл текущего слова.

---

### Hidden State

Внутренняя память RNN.

---

### Attention

Механизм, позволяющий модели выбирать наиболее важные элементы входной последовательности.

---

### Self-Attention

Attention внутри одной последовательности.

---

### Transformer

Архитектура, построенная вокруг механизма Self-Attention.

---

### Encoder

Часть Transformer, предназначенная для понимания входного текста.

---

### Decoder

Часть Transformer, предназначенная для генерации последовательности.

---

### Tokenizer

Алгоритм преобразования текста в последовательность токенов.

---

# Что важно понимать

Не пытайтесь запоминать архитектуры по отдельности.

Гораздо полезнее понимать, **почему появилась каждая следующая модель**.

| Архитектура | Какую проблему решила? |
|-------------|------------------------|
| One-Hot | Представление слов числами |
| Bag of Words | Учёт частоты слов |
| TF-IDF | Учёт информативности слов |
| Word2Vec | Семантическая близость слов |
| RNN | Учёт порядка слов |
| LSTM | Длинные зависимости |
| Attention | Потеря информации в Seq2Seq |
| Transformer | Последовательные вычисления |

Если вы можете воспроизвести эту таблицу по памяти, значит главная идея главы усвоена.

# Частые ошибки

### ❌ "Токен — это слово."

Нет.

Современные модели обычно используют подсловную токенизацию.

---

### ❌ "Word2Vec понимает предложения."

Нет.

Он строит эмбеддинги отдельных слов.

---

### ❌ "LSTM полностью решила проблему длинного контекста."

Нет.

Она лишь значительно уменьшила влияние этой проблемы.

---

### ❌ "Attention и Transformer — одно и то же."

Нет.

Attention — это механизм.

Transformer — архитектура, использующая этот механизм.

---

### ❌ "BERT и GPT отличаются только размером."

Нет.

Это модели с разной архитектурой и разными задачами обучения.

# Production Checklist

Перед тем как использовать NLP-модель в production, полезно ответить на несколько вопросов.

### Если нужна классификация текста

Обычно используют Encoder-only модель.

Например, BERT.

---

### Если нужна генерация текста

Используют Decoder-only модель.

Например, GPT или Llama.

---

### Если требуется перевод или суммаризация

Часто используют Encoder-Decoder модель.

Например, T5.

---

### Если требуется поиск по документам

Чаще всего используются Encoder-модели для построения эмбеддингов.

---

### Если модель не помещается в память

Используют квантизацию, дистилляцию или более компактные архитектуры.

Подробно эти темы мы рассмотрим в следующей главе.
